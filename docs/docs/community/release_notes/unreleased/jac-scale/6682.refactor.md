- **Refactor: shared "no-serve startup" seam for in-process runtimes** (issue #6682): the module-load + server-construction + endpoint-registration core that `ScaleTestClient` and the Jac-native desktop host (#6596) each hand-copied out of `JacAPIServerCore.start()` -- and had already drifted on -- now lives in one place, `jac_scale.inprocess.prepare_inprocess_runtime(jac_file, base_path, endpoint_profile, setup_background_services, transport_kind) -> PreparedInProcessRuntime`. It prepares a `port=0` `JacAPIServer` (no socket bind), registers an endpoint profile (`"desktop"` / `"test"` / `"full"`), creates the guest user, installs the shared-root resolver explicitly (so `root.shared` no longer depends on construction order), idempotently registers the jac-scale plugin, and builds the transport. `transport_kind` is currently `"testclient"`; `"asgi"` / `"isolated_worker"` are reserved so the execution-isolation boundary lives in the seam API. `ScaleTestClient._initialize()` and the desktop `inprocess_dispatch.boot()` now both call it instead of duplicating ~30-40 lines of registration each.
