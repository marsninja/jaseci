- **Docs: the Kubernetes deploy docs no longer describe the removed image-build pipeline**: the Docker image-build subsystem (build, registry push, and local-cluster image loading) was deleted and the copy-source "no-image" path became the only one that ships, but no docs were updated. The reference docs, the Kubernetes tutorial, the CLI reference, the internals inventory, and the shipped `jac-sv-deploy` skill all still advertised a `--build` / `--registry` flag, `image_registry` / `docker_image_name` config keys, `DOCKER_USERNAME` / `DOCKER_PASSWORD` credentials, and `kind load docker-image` / `k3d image import` / `minikube docker-env` -- none of which exist. Those sections now describe what actually happens: no image is built and no registry is needed on any cluster, because the source ships into the cluster as a bundle on a PVC and pods boot from a stock base image (override with `python_image`). The removal is recorded in Breaking Changes, and a docs test now fails if the removed surface reappears in docs that describe current behavior.
