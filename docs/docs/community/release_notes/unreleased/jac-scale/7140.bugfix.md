- **Bugfix: scale deploys of projects with no `[plugins.scale.*]` section no longer crash-loop on pymongo** (issue #7137): the `data.mongo` default for a scale serve was gated on the intent dict being non-empty, so exactly the projects with no declared scale config missed it. The gate is now keyed on the scale/serve context instead of intent truthiness, and the deployer records the decision it acts on: when Kubernetes provisioning enables MongoDB and the project declares no database backend, the shipped bundle's `jac.toml` gets the matching `[plugins.scale.database] backend = "mongo"` (or `[scale.database]` when the project uses the `[scale]` form) baked in, so the pod's `jac install` closes over `data.mongo`. Deploy tooling is also now synced automatically: creating a Kubernetes deployment target installs any missing `deploy.k8s` dependencies (kubernetes/docker, pinned by the jac binary's capability registry) into the jac-owned global site, so a scale deploy no longer requires a manual `jac install --scale` on the operator machine.
