- **Refactor: Capability-oriented dependency guards**: jac-scale's optional-dependency guards now source their error messages from the core capability registry. `require_optional` takes a capability name (e.g. `data.mongo`, `deploy.k8s`) instead of a pip-extra group, and every `pip install jac-scale[...]` hint in the guards and runtime diagnostics is replaced with capability-oriented guidance. The old per-shim extras taxonomy is gone.
