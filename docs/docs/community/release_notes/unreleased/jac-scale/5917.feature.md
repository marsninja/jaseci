- **Feat: MongoDB server-side type and field query pushdown**: `MongoBackend.query()` now accepts an optional `mongo_filter` dict that is pushed to `collection.find()` before the Python post-filter, eliminating full collection scans. New `query_by_type(arch_type_name, mongo_filter)` pushes both the archetype type and optional field predicates (e.g. `{'age': {'$lt': 28}}`) to MongoDB, auto-prefixing field names to `data.archetype.*`. `get_roots` now uses `{'arch_type': 'Root', 'type': 'NodeAnchor'}` instead of a full scan. `ensure_indexes()` (called at startup) creates a compound `(arch_type, type)` index so all type-based queries use IXSCAN instead of COLLSCAN.
