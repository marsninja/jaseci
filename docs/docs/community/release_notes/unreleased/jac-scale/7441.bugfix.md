- **Fix: `jac start --scale` no longer dies on `No module named 'dotenv'`**: the Kubernetes realizer imported `dotenv` at module scope, so a clean operator machine crashed before the `deploy.k8s` auto-install had a chance to run, right after jac announced it was installing the deploy tooling. The deploy entry points (`realize`, `destroy`, `status`) now sync the `deploy.k8s` closure before their first third-party import, the dotenv imports moved to their point of use, and an unsynced `--dry-run` (which by design installs nothing) fails with actionable capability guidance instead of a bare `ModuleNotFoundError`. A dry run on a machine without the tooling still renders manifests when no `.env` is present.
