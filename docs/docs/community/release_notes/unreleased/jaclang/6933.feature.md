- **Feature: native `__class__.__name__` / `type(self).__name__`**: both forms now lower in native (`na` / `.na.jac`) code instead of raising `E5090` (#6930 workstream A2). The result is runtime-correct: a class with a vtable reads its name from the instance's vtable, so an inherited base method reports the concrete subclass name, while a vtable-less class lowers to a constant string load. This also fixes a type-checker bug where any `@<prop>.setter` / `.getter` / `.deleter` decorator raised a bogus `E1031` ("Cannot access attribute 'setter'") under type checking, because a `@property` getter resolved to a function rather than a property object.
