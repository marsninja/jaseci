- **Refactor: Edge-traversal resolution folds hop-by-hop with no global fallback**: `refs()` walks `ObjectSpatialPath` chains through a new `jaclang.runtimelib.resolver` whose `resolve_path_anchors` calls a uniform per-hop `resolve_hop` at every position. Each hop locally picks index narrowing (when the hop has no predicate filters, has a root anchor, and the input IDs are all owned by that root) or a single-hop `edges_to_nodes` walk fallback (for predicates, foreign-root IDs, missing index, transient origins). The all-or-nothing strategy decision and slow-loop fallback in `planner.execute_path` are gone: `resolve_path_anchors` always returns a `list[NodeAnchor]`, and `refs()` no longer carries a global slow-loop tail. Resolves the fast/slow split described in issue #6778. `planner.execute_path` becomes a one-line wrapper around `resolve_path_anchors` to preserve direct test imports in `test_topology_e2e.jac`. Edge-only queries walk their prefix through the same per-hop fold and then run the existing edge terminator.
