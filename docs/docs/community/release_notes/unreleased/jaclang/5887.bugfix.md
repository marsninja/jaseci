- **Fix: type-checker validates `visit` collection element type**: `visit` accepted any `list`/`tuple`/`set`/`frozenset` target unconditionally, so `visit ["a", "b"]` and `visit [1, 2, 3]` slipped through with no diagnostic. The visit handler now recurses into the collection's element type(s) and emits `E1094` when an element is not a node, mirroring the connect-operand validation. Unparameterized collections still pass.
