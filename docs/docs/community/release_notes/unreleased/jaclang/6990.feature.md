- **Feature: native C-floor archives are bundled into the payload so installed binaries can static-link them (#6978 Phase 0.2)**: `nacompile`'s static C-floor archive linking (Phase 0.1a) previously resolved archives only from the dev `.pbs-build` tree, so a shipped binary had nothing to link and silently fell back to a dynamic dylib dependency. `mkpayload` now stages a curated floor set (`lib{ssl,crypto,sqlite3,mpdec,lzma,bz2,expat,z,zstd}.a`, the bundled-C set the native stdlib roadmap targets; best-effort per file so macOS's absent `libz` is skipped) into `python/floor/<osarch>/`, plus certifi's `cacert.pem` into `python/floor/cacert.pem` for the future ssl floor. The `nacompile` locator (`_floor_lib_dir`, split into a pure `_resolve_floor_dir`) now resolves either the dev `.pbs-build` tree or the installed `python/floor/<osarch>` payload layout, arch-keyed so a cross-`--target` build never links the wrong arch; `JAC_NATIVE_FLOOR_DIR` and `JAC_NATIVE_CA_BUNDLE` override the lookup.
