- **Python-style properties are nudged toward native Jac syntax**: `@property` and the same-object `@x.getter` / `@x.setter` / `@x.deleter` now report `W2029` on every compile, suggesting the native `has x: T { getter; setter(value); ... }` form. This brings them in line with `@staticmethod` (`W2007`) and `@classmethod` (`W2006`), which were already reported this way; previously `@property` was only flagged under the opt-in `jac check --lint`. The Python-compat `class` archetype stays exempt, and `jac check --lint --fix` rewrites the decorator form automatically.
