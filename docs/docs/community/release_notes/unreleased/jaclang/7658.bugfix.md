- **Fix: native topology store scales O(degree) with region-scoped death, walker engine and RC leaks repaired**: the native OSP edge store was a global table scanned linearly on every `[-->]` (cost grew with every edge ever created; 58μs/visit at 100k edges) whose rows outlived region death, leaking footprint and letting recycled addresses resurrect dead topology. Edges now live in a per-node adjacency index (hash-indexed chains over flat int arrays, backward-shift key deletion), `osp_conn` records the endpoints' region via the new `__jac_region_handle_of` primitive, and region drop retracts all of a dead region's rows before the bulk free, so topology metadata dies with its topology and service footprint plateaus instead of growing without bound. The walker engine dequeues via cursor instead of `pop(0)` and no longer rebuilds the visit queue or boxes phase tuples per visit. Native RC codegen leaks are fixed across list slice/concat/repeat/`pop` results, tuple destructuring (including through a bitcast that dropped ownership), `[-->]` result lists, spawn start-lists, and owned `insert` arguments. A 200-node × 500-request build-traverse-discard service drops from 5.81s/184MB to 19.2ms/6.1MB with all 100k `drop` hooks firing, 20k walker spawns from 1.75GB leaked to 23.6MB bounded, and a million-object region move costs 1.02μs against 789ms for deep copy, with the `region_of` growth-rule branch at allocation parity (24 vs 23 ns).
