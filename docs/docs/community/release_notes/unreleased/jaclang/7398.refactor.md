- **Refactor: native RC header returns to 32 bytes; per-object trace pointer removed (#7398)**: The cycle collector in #7149 grew the native object header from 32 to 48 bytes by adding a trace-function slot and a color slot. Both are removed: the collector's color and buffered bits now pack into spare high bits ([50:48]) of the existing alloc-id header word (magic occupies [47:32], and the magic check masks accordingly), and per-type trace functions are resolved at collection time through a compile-time-generated `__rc_trace_lookup` keyed on the destructor pointer already stamped in the header (type-unique for both structs and containers) instead of a per-object slot. The header is 32 bytes again, so every field offset shifts back -- a native-ABI change, and all native artifacts must be recompiled together (there is no cross-version header compatibility). Behavior of the collector is otherwise unchanged.
