- **Feature: fully-markerless co-location in `pages/`**: the file-based route scanner now decides route membership from the top-level symbols a module defines (a route defines `page`, a layout `layout`) instead of a filename skip-suffix. A markerless co-located component that defines its own name (e.g. `budget_ui.jac` defining `BudgetUI`) can sit beside its route in `pages/` without being auto-registered as a broken route. The signal is the compiled symbol table (`sym_tab.names_in_scope`), so it catches both `def:pub page` and a plain `def page` inside `cl {}` (the pub-gated client export list does not). When a page cannot be resolved the scanner keeps it as a route rather than dropping it, so a transient compile error never silently removes a route. `.impl.` and `.test.` sidecars are still skipped structurally as non-route module roles; the placement marker `.cl.` is no longer needed or consulted (legacy `.cl.jac` components keep working because they define a non-`page` symbol). Closes #7640.
