- **Desktop Plugin IPC: OS capabilities for webview apps**: Desktop apps now expose a plugin host with seven built-in plugins (`path`, `window`, `fs`, `notification`, `clipboard`, `dialog`, `shell`) to JS via `window.__jac.invoke()` and `window.__jac.on()`, with per-plugin security gates (allow-lists, opt-in toggles, and deny-all shell defaults) configured under `[plugins.desktop.plugins]` in `jac.toml`. A typed `@jac/desktop` client SDK, plugin manifest, and registry ship alongside; the native host bootstraps the built-in plugins at startup and transpiles plugin `.jac` sources at build time. A `desktop-demo` example exercises notification IPC, and plugin build failures now print the full Python traceback.
- **Client Compiler: `@jac/*` modules resolve via entry points**: Packages can register virtual `@jac/*` modules through the `jac.modules` entry-point group so `@jac/desktop` and similar SDKs resolve without vendoring into jaclang. The Vite bundler now compiles a resolvable `@jac/desktop` SDK to `compiled/desktop_api.js` and wires the alias when present.
- **Native Compiler: SubTag type annotations lower correctly**: Native `_resolve_jac_type` unwraps `SubTag` wrappers so annotated globals and clib fields (e.g. `Callable[...]`) resolve to the correct LLVM types without re-reading `type_tag.tag`.
