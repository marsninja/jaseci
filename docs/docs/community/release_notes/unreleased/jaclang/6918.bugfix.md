- **Fix: `jac nacompile --shared` initializes module-global clib structs on load (#6917)**: `_inject_shared_init` collapsed its own-module glob-init detection to a single unquoted substring scan (`"@__jac_glob_init("`), but LLVM quotes the symbol -- the emitted IR is `define void @"__jac_glob_init"()` -- so the scan missed it and `__jac_shared_init` was generated empty. The global's initializer never ran on `dlopen`, the struct-pointer slot stayed NULL, and the first field write/read into a global clib struct segfaulted (the desktop-cef target died here in `cef_dispatch_settings_create` before `cef_initialize`). The same code built as a plain executable was always correct: the JIT path resolves the init via `get_function_address`, immune to the text scan. The detection now checks both the quoted and unquoted forms, matching the transitive-source branch. Added a unit test on the production emitter (the quoted-symbol guard) and an end-to-end `--shared` global-clib-struct fixture + dlopen test; the shared-lib test helper was rerouted through the real `_inject_shared_init` so the suite exercises the production injection path instead of a divergent copy (the divergence is what let this slip past the whole file).
