- **Fix: `jac check` / `jac lint` no longer emit spurious `LiteralString` type errors on files after the first**: releasing each file's compile closure between files drops and recreates the type evaluator, which used to re-mint the synthesized `LiteralString` type with a fresh identity every file while the stub AST it derives from still cached the prior instance -- breaking `Literal[...] <: LiteralString` and surfacing false `E1001`s on every checked file after the first. The synthesized type is now memoized on the stable `str` class so it survives evaluator churn.
