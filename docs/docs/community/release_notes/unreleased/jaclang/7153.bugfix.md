- **Fix: `get_object`/`aget_object` now enforce read-access permissions**: resolving an object by id previously skipped access validation entirely, so the `&(id)` deref operator, the `jobj` builtin, and the server's `_jac_spawn_node` walker-target resolution could all reach (and spawn walkers on) nodes owned by other roots given a known id. Both lookups now run `check_read_access` on the fetched anchor and return `None` when access is denied, indistinguishable from a missing object. `get_shared_root` fetches through memory directly since it legitimately resolves a still-`NO_ACCESS` root before granting `CONNECT` on it.
- **Fix: `reset_graph` crashed on the sqlite backend**: it still read anchor rows with `pickle.loads`, but the sqlite store moved to JSON payloads long ago, so any `reset_graph()` call against a persisted graph raised `TypeError: a bytes-like object is required, not 'str'`. It now selects the anchor id and owning root via `json_extract` and deletes through the memory layer.
- **Fix: enum fields on nested objects lost their type after a persist/reload round-trip**: enums serialize as their raw value, and while archetype fields are coerced back through their declared types on load, plain nested objects (an `obj` stored inside another `obj`'s field) were rebuilt with raw `setattr` — so `enum_field` came back as `'b'` instead of `Enum.B`. The generic object deserializer now coerces each field through its declared type, matching archetype behavior.
