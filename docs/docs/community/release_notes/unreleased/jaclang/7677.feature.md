- **Feature: markerless shared logic follows its client importers into the browser**: a plain (non-`sv`) client import of a markerless `.jac` module now places the imported accessless symbols in the client codespace under the same policy as the intra-module pull: accessless defs, globs, and one-sided objs relocate; `def:pub` stays a server endpoint and bridges over RPC; OSP archetypes never move. Pulled elements dual-emit (python emission is kept), so the module remains importable from server code, and their names join the module's ES export surface. A new `W6005` warns when a client call to a `def:pub` endpoint passes a function-typed argument, which cannot cross the RPC boundary. Separately, a `pages/` tree that yields zero recognized routes now emits a console error naming the blank-page outcome, with a per-file hint for exports returning `JsxElement` that should return `JsxPage` or `JsxLayout`.
