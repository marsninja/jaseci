- **Fix: `py2jac` handles implicit string concat across a backslash line-continuation (#6928)**: Converting Python where adjacent string literals were joined by an explicit `\` line-continuation (`"abc" \` ⏎ `"def"`) leaked the source backslash into the emitted Jac, producing unparsable output, and `py2jac` exited `0` with empty stdout, silently dropping the file in batch ports. `PyastBuildPass` now strips line-continuation backslashes that sit outside a string literal while preserving those inside one (and triple-quoted multilines), and `jac py2jac` exits non-zero when the conversion produces errors.
