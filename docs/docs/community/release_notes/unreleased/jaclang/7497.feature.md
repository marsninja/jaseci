- **Sharper `jac tool grammar` extraction and a new `--strict` diagnostics mode**: the EBNF extractor now computes FIRST sets across rules so dispatch guard tokens consumed by the called rule are no longer double-counted, emits the full token set for `or`-merged `match_tok` conditions instead of only the first token, renders opaque if/else branches as alternations rather than sequences, and collapses a bare choice when its optional/star twin is present. `jac tool grammar --strict` reports parser constructs invisible to extraction (bare `advance()` calls, token methods called with non-literal arguments, and `self.*` methods with no declared grammar semantics), and `jac.spec` has been regenerated with the improved output.
