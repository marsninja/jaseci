Added `jac ninja`, a fully fused terminal editor built into the jac binary. A hash-pinned Neovim 0.13 fork (jaseci-labs/neovim) is statically linked into the launcher itself (no external editor, no separate process image) and boots in milliseconds without starting Python; the runtime files, the bundled tree-sitter parsers (including the jac grammar and python for inline `::py::` blocks), and an opinionated jac-only configuration ride in the existing payload. Out of the box: tree-sitter syntax highlighting and folds, full LSP (completion, hover, go-to-definition, rename, diagnostics) served by `jac lsp` from the same binary, fuzzy finding, live grep, a file explorer, `space-j` keymaps that run/test/check the current file, and in-editor agent orchestration: `space-a` drives `jac ai` (the binary's own coding agent) in managed sessions with ask/explain/fix-diagnostics actions. The editor is hermetic (a user's own Neovim config is never read) and its state lives under `~/.local/state/jac-ninja`. Builds can opt out with `-Dno-ninja`.
