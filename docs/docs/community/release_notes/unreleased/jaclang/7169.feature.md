- **`jac ninja`: a fused terminal editor built into the jac binary**: a hash-pinned Neovim 0.13 fork (jaseci-labs/neovim) is statically linked into the launcher itself and boots in milliseconds without starting Python; the runtime files, bundled tree-sitter parsers (including the jac grammar, plus python for inline `::py::` blocks), and an opinionated jac-only configuration ride in the existing payload. Out of the box: tree-sitter syntax highlighting and folds, full LSP (completion, hover, go-to-definition, rename, diagnostics) served by `jac lsp` from the same binary, fuzzy finding, live grep, a file explorer, `space-j` keymaps that run/test/check the current file, and in-editor agent orchestration (`space-a` drives `jac ai` -- the binary's own coding agent -- in managed sessions with ask/explain/fix-diagnostics actions). A VSCode-style easy mode (`jac ninja --easy`, persistent, `:NinjaEasy` to toggle) provides CUA keybindings, shift-arrow selection, and insert-first buffers for users new to modal editing. The editor is hermetic (a user's own Neovim config is never read), state lives under `~/.local/state/jac-ninja`, and builds can opt out with `-Dno-ninja`.
