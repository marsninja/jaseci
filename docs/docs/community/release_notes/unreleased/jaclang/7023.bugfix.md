- **Fix: native `open(path, flags)` no longer ICEs the precompile**: A native module that calls the POSIX `open(path, flags_int)` syscall while its libc `open` extern is out of scope (e.g. a standalone bundled-payload precompile of a module whose imports live in a sibling file) no longer crashes the native backend with an uncaught `TypeError: Type of #2 arg mismatch: i8* != i64` from the LLVM IR builder. Because `open` is also the file-open builtin (`fopen(path, mode_str)`), an integer second argument was being forced through `fopen`; the backend now detects the non-string mode and emits the same clean `E5090` every other unlowerable native call gets. The file-open builtin (`open(path)`, `open(path, "r")`, `open(path, "rb")`) and a declared libc `open(path, flags) -> i32` extern are both unaffected. Clears the `✖ Error: FAIL` on `cef_platform.na.jac` (#7010 Error 2).
