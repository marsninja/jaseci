- **Fix: bundle-loader "Permission denied" on any non-world-writable volume**: the jac-scale bundle-loader pod relied on `securityContext.fsGroup` to make the bundle PVC writable, but kubelet never applies `fsGroup` to `hostPath` or NFS volumes, and CSI drivers with the default `fsGroupPolicy` skip `ReadWriteMany` volumes -- exactly what the bundle PVC must be. Deploys onto anything other than world-writable storage (a `hostPath` PV on kind, NFS exports, most RWX CSI drivers) aborted at `mkdir: Permission denied`. The loader pod now runs a hardened `bundle-perms` init container that chowns the volume root to the loader user before seeding, so any RWX backend works without out-of-band `chmod 777` workarounds; storage that squashes root (for example EFS access points) keeps working as before.
