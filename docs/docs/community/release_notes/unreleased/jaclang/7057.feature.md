- **Feature: bundled native `socket` (TCP client) over a libc BSD-sockets floor (#6978 Phase 1)**: `import socket` resolves on the native (`nacompile`/JIT) pathway to a bundled `na_stdlib` module -- a pure-Jac TCP-client surface (`socket.na.jac`) over a thin libc FFI floor (`_socket_native.na.jac`). Unlike the `hashlib`/`secrets` floors that statically link a bundled `libcrypto.a`, the socket primitives are plain BSD sockets in libc (`socket`/`connect`/`send`/`recv`/`getaddrinfo`), so a socket program links nothing beyond libc/libm and is self-contained with no bundled archive -- and no libcurl. Coverage is a blocking TCP client: the `socket(family, type, proto)` object with `connect`/`send`/`sendall`/`recv`/`close`/`fileno`, the family-spanning `create_connection((host, port))` helper that the `ssl` wrapper and `http`/`urllib` build on, and the `AF_*`/`SOCK_*` constants; setup and transport failures raise a catchable `OSError`. A new equivalence suite (`test_socket_equivalence`) pins the native client congruent with CPython's `socket` against a loopback server (HTTP status + exact byte count) and asserts that connecting to a closed port raises a catchable `OSError`. This is the transport `ssl` and `http`/`urllib` are built on (#7052). DNS resolution walks `getaddrinfo`'s `addrinfo` list with no new compiler intrinsic -- a libc `memcpy` whose source pointer is carried as a plain Jac `int` (the `FILE*`-as-`int` FFI convention) copies the sockaddr out and `int.from_bytes` reads the fields.
