- **Scale: pod dependency installs survive the official base image**: the bootstrap now exports `PIP_PREFIX=/app/.jac/venv` so `jac install` lands capability and app dependencies in the project venv on the shared volume instead of the runtime cache site (a container-local layer on `jaseci/jaclang` images, where an init container's installs evaporated before the main container started). Non-mongodb capabilities like `data.redis` and `llm.mcp` no longer crash-loop served pods.
