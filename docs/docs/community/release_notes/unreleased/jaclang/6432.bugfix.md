- **Fix: native `dict.pop` and `complex` codegen**: `dict.pop` no longer emits a malformed LLVM PHI node on the key-missing path that raises `KeyError` (the runtime-raise splits the block, so the real predecessor is now recorded). `complex` now works end to end: the `complex` annotation lowers to a boxed `JacComplex*` instead of falling through to `i64`, so `.real`/`.imag` and `+`/`-`/`*` arithmetic return correct values instead of `0.0`.
