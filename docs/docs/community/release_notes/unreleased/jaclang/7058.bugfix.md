- **Fix: `nacompile` links bundled `na_stdlib` imports for auto-promoted `.jac` files**: a regular `.jac` (no `na {}` block) that imports a bundled native module is auto-promoted to the native backend, but `BoundaryAnalysisPass` only classified the import as a native cross-module import when the file was a `.na.jac` or the import sat in a `na {}` block. The auto-promote case skipped the cross-module IR merge, so the entry point called each module's `__jac_glob_init_<safe>` against an undefined symbol and the binary aborted at load. The auto-promote target is now treated as a native context, so the imported module's IR (and its initializers) is merged into the binary.
