- **Feature: native codespace is inferred from extern-decl imports - fully markerless three-codespace modules**: building on the client-codespace inference in #7578, an import whose braces contain C-ABI function declarations (`import from raylib { def InitWindow(w: i32, h: i32, title: str) -> None; }`) declares an FFI surface only the native backend can satisfy, so it now seeds native placement, and the declarations using those extern names (plus the globs, objs, and helpers they reference) follow through the same symbol-graph propagation. Consuming a native *module* by name is deliberately not a native signal (that remains the server-to-native interop crossing), and native-compatible pure code still defaults to server: compatibility is not intent. Propagation is two-colored with a hard/soft split - elements using seed-declared names (npm items, extern decls) take that color unconditionally, then referenced-by dependencies follow, and a declaration referenced by both the client and native sides stays on the server where both can bridge to it. The Python backend prunes inferred-native declarations from mixed modules (keeping `.na.jac` files' legacy projection intact), and the client build's wasm step now triggers off the compiled module's declared codespaces instead of regex source-sniffing. A fully markerless module can span all three codespaces: validated end-to-end on the raylib shooter showcase, whose markerless form compiles to LLVM IR identical to its `.na.jac` original and whose entry module (server import + JSX client app + extern-import game, zero markers) builds the same wasm artifact as the `na {}`-block version.
