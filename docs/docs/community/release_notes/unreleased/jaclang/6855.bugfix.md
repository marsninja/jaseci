- **Fix: jaclang-internal type stubs leaking into the user program's module hub**: `_resolve_program` now resolves symlinks (`os.path.realpath`) on both the jaclang root and the candidate module path before deciding whether a file is a jaclang-internal stub (typeshed/builtins `.pyi`) or user code. Previously it used `os.path.normpath`, which does not resolve symlinks, so when jaclang is materialized under a symlinked directory (e.g. the single binary's cache, or macOS `/var` -> `/private/var`) the internal stubs were routed into the user program's `mod.hub` instead of the compiler's internal hub. This also lets the test suite run unchanged through the single `jac` binary.
