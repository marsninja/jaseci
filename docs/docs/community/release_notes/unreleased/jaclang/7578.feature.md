- **Feature: client codespace is now inferred - `cl {}` blocks and `.cl.jac` extensions are optional**: the compiler derives each declaration's codespace from the code itself. Elements carrying structurally client-only syntax (JSX, string-path npm/asset imports) seed CLIENT placement at parse time, and the new `CodespacePullPass` propagates it through scope-aware symbol references - helpers, globs, and imports used by client code join the client bundle automatically, while `def:pub` endpoints, walkers, and shared archetypes stay on the server behind their existing auto-RPC/auto-share bridges. Native placement is inferred too, from extern-decl (C-ABI FFI) imports such as `import from raylib { def InitWindow(w: i32, h: i32, title: str) -> None; }` - an FFI surface only the native backend can satisfy - with the same reference-based propagation, so a fully markerless module can span all three codespaces; consuming a native *module* is not a native signal (that stays the server-to-native interop crossing), a declaration referenced by both the client and native sides stays on the server where both can bridge to it, and native-compatible pure code still defaults to server. Explicit markers (`cl {}`/`sv {}`/`na {}`, prefixes, file suffixes) remain as overrides that inference never touches and that never act as propagation sources - an explicitly marked region is an author-drawn boundary. `sv import` semantics were also cleaned up along the way: the `sv` marker is now purely a boundary fact (the target lives server-side) while the import's own placement is inferred from its consumers, so client-consumed sv imports become RPC stubs and server-consumed ones remain microservice calls - and import classification itself (npm vs python vs jac vs `@jac/` virtual, sv-marked or not) now has a single source of truth as computed getters on the Import node. Fully markerless client apps compile to byte-identical JS versus their `.cl.jac`-marked equivalents.
