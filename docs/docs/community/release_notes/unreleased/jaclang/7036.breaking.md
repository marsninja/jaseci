- **Breaking: Project kinds renamed to deliverable-oriented names**: The `jac create --kind` / `[project] kind` values have been renamed for consistency: `native-app`→`cli-native`, `shared-library`→`native-lib`, `api-service`→`service`, `microservices`→`service-mesh`, `pypi-package`→`py-package`, `npm-package`→`js-package`, `fullstack`→`web-app`, and `client`→`web-static` (`cli`, `native-binary`, `desktop`, `mobile` are unchanged). The client framework's built-in templates were renamed to match (`--use fullstack`→`--use web-app`, `--use client`→`--use web-static`). There are no back-compat aliases, so existing `jac.toml` files that set an old `kind` value (or scripts that pass an old `--kind`/`--use` name) must be updated.
