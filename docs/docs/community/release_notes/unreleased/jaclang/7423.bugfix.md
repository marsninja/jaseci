- **Fix: `jac run` on desktop and mobile projects**: leftover plugin gates blocked two project kinds that core already supports. `jac run` in a `desktop` project refused to start, reporting that the kind is run via a plugin whose name printed as blank, and `jac run` in a `mobile` project silently built a default `.jab` artifact instead of the mobile client shell. Both kinds now dispatch through the `--client` target that `jac start` and `jac build` already accept, and `desktop` is accepted by `--client` on both commands rather than only advertised in their help text.
