- **Enable type checking by default for codegen paths**: `JacProgram.compile()` previously defaulted to `type_check=False`, silently skipping type checks on `jac run` / `jac build` / `jac2py` / `jac2js`. The parameter now auto-infers from `no_cgen` -- codegen compiles type-check, symbol-table-only loads (annex pass, import resolver) skip as before. Includes companion structural moves: `MTIRGenPass` runs after `TypeCheckPass` so it can read evaluator-populated `.type` instead of walking AST text, and the duplicated primitive-name lists in `mtir_gen_pass` and `unitree_enrich_pass` now derive from `JAC_TYPE_REGISTRY` (single source of truth). `.na.jac` CLI execution keeps `type_check=False` pending closure of native-pathway gaps the central evaluator doesn't yet model.
