- **Build: pure-Zig payload tool (no bash/curl/git/zstd/tar)**: the single-binary payload build's shell scripts (`fetch-pbs.sh`, `fetch-typeshed.sh`, `mkpayload.sh`, `update-typeshed.sh`) are replaced by one std-only Zig tool (`launcher/payload.zig`) that does the HTTP download, checksum verify, (de)compression and tar in Zig's `std`, shelling out only to the bundled CPython for `pip` and the JIR precompile. Build-time host dependencies drop to just `zig` plus a network connection (and an optional, best-effort `strip`). The payload's inner compression switches zstd to gzip so both ends are pure std, the typeshed stubs are fetched over HTTP and verified against a pinned `TARBALL_SHA256`, and the bundled libpython is now stripped, so the shipped binary is slightly smaller than before despite gzip's lower ratio.
