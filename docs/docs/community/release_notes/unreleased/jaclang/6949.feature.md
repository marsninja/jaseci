- **Feature: Native stdlib `urllib.request` over a libcurl floor + variadic FFI (#6940 Phase 3)**: the native (`na`) pathway can now fetch over HTTP. A bundled `na_stdlib/urllib/request.na.jac` exposes `urlopen(url)`, which performs a real HTTP/1.1 GET -- DNS, TLS, and redirect following -- over **libcurl** reached through the native FFI (**Mechanism F**), not a reimplemented transport stack: libcurl abstracts the per-OS TLS backend (SecureTransport / OpenSSL / Schannel), so one binding covers every native host. The easy handle streams the response body and headers to temp `FILE*`s via libcurl's default `fwrite` callback (`CURLOPT_WRITEDATA` / `CURLOPT_HEADERDATA`), which the module reads back with the native `open().read()` builtin and parses into an `HTTPResponse` record (status, body length, reason, url, body, headers). Enabling this, the native FFI gained **variadic C declarations**: a trailing `*rest` param on a clib `def` (e.g. `curl_easy_setopt(handle, option, *rest)`) marks the symbol variadic so the backend emits a `var_arg` declaration over the fixed params -- required because on AArch64/AAPCS a variadic argument must pass on the stack, where a fixed declaration would wrongly pass it in a register and the value never reaches the callee. Pinned sv<->na congruent by the new `test_urllib_equivalence.jac`, which stands up a loopback `http.server` (no live network) and compares the bundled native `urlopen` against CPython's real `urllib.request` on HTTP status and received body length, for a direct GET and a 301 redirect both follow. Native-host-only (clean E5090 on wasm); partial coverage per #6940 (v1 is GET + redirects; `Request`/`Range`, POST, `timeout`, response accessor methods, and `HTTPError` on 4xx/5xx are documented next slices). Extends #6404; follows #6941.
