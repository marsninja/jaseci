- **Refactor: remove `jac bundle --precompile`**: The shipped `_precompiled/*.jir` bundle is a cold-start optimization, not a correctness requirement; the runtime lookup is version-keyed (`cpython-3XX`) and opportunistic, so a wheel built without it is fully valid and recompiles on first import. For PyPI plugin wheels the benefit only applied to consumers on the exact build-time Python and bloated an otherwise portable wheel, so the `--precompile` flag (and `_precompile_project`) are dropped from `jac bundle`, along with their use in the release workflows and release scripts. The native binary's own precompile path (`mkpayload`) is unchanged. With the flag gone, `find_package_dir` in `precompile_bytecode.jac` collapses to its single live pass (the `__init__.py` lookup); the namespace-package and last-resort passes were dead code (the former targeted the removed `jac-client`).
