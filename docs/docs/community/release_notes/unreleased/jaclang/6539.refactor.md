- **Refactor: Extract FrameworkBackend seam from EsastGenPass**: Client codegen (cl.jac / cl {} → JavaScript) is no longer hard-wired to React. A new pluggable `FrameworkBackend` interface and framework-neutral reactive-intent vocabulary (`StateField`, `RefField`, `StateUpdate`, `Effect`, `AsyncBoundary`) allow per-framework lowering. All React-specific emission (`useState`, `useRef`, `useEffect`, `setCount`, `<JacAwaiting>`, `<JacClientErrorBoundary>`) moves into `ReactBackend`. A `[client] framework` config section (default `"react"`) selects the backend at compile time. Unknown framework names now emit `E5081` through the EsastGenPass diagnostic path instead of surfacing as a bare `ValueError`.
- **Feature: Preact backend (Phase 1)**: `framework = "preact"` now selects `PreactBackend`, which reuses React's hooks lowering but imports `useState`/`useEffect`/`useRef` from `preact/hooks`, compiles a Preact-backed `@jac/runtime`, and emits a `render`/`h` entry script.
- **Refactor: Split `client_runtime_core` from the React shim**: Framework-agnostic `@jac/runtime` symbols (auth, walker/function fetch + auto-cache, localStorage, navigation, error reporting) move into `client_runtime_core.cl.jac`; `client_runtime.cl.jac` becomes a thin React/Preact shim that re-exports the core surface so `@jac/runtime` stays one module. The Vite compiler emits the core sibling alongside the shim.
- **Feature: TanStack Form for React and Solid runtimes**: `useJacForm`/`JacForm` now use `@tanstack/react-form` and `@tanstack/solid-form` (Standard Schema validators) instead of `react-hook-form`/`@hookform/resolvers` and `solid-hook-form`. The hand-rolled `__zodResolver` is removed; Zod schemas validate natively through TanStack Form's `validators` API. Peer dependencies, default client deps, and npm publish maps are updated accordingly.
