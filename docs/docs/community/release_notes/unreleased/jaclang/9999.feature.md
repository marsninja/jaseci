- **Reactive `has` fields compile to Preact signals**: Client-side `has` fields now lower to `useSignal` instead of `useState`, and `can with entry` lowers to `useSignalEffect`. Reads of a reactive name are rewritten to `name.value` everywhere they appear (conditions, JSX, arguments), and writes become `name.value = ...`. This eliminates the stale-closure class of bugs that could previously surface when an effect body read a `has` field after writing to it. Explicit dependency lists on `can with [a, b] entry` are now ignored -- signal reads inside an effect body auto-track -- and the compiler emits a new `W5033` diagnostic pointing users at the deprecation. `can with exit` continues to compile to `useEffect(fn, [])` so cleanup-at-unmount semantics stay uncoupled from signal-driven re-runs.
