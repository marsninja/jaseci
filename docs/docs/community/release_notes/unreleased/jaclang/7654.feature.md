- **Feature: a client route is now a type, not a name**: `JsxPage` and `JsxLayout` are ambient builtin types (no import, like `JsxElement`). A `pages/` module is a route when a public export returns `JsxPage`, and a layout when one returns `JsxLayout`; the export name is free and the generated router imports it by that name (`def:pub Dashboard() -> JsxPage` serves `pages/dashboard.jac` at `/dashboard`). A co-located component returns `JsxElement`, so it is simply not a route, with no marker or filename rule. The filename still decides the URL (`index.jac`, `[id].jac`, `[...rest].jac`, `(auth)/` groups); only membership moved to the type. `JsxElement` is assignable to both markers, so a component body returning JSX satisfies either signature while the annotation stays distinct, and a bad annotation is a type error rather than a silently missing route. Route components are emitted as JS exports regardless of `:pub`, so a non-pub page inside a `cl {}` block still resolves; when a page fails to compile the scanner falls back to its parsed return-type annotation so the route survives instead of vanishing from the table. **Breaking:** defs named `page` / `layout` no longer mark anything on their own - existing pages need `-> JsxPage` / `-> JsxLayout` added.
