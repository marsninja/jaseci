- **`jac check`: lower peak memory.** The per-file check loop now drops the duplicate per-token terminal stream retained on each module (only the formatter reads terminals, not check) and calls glibc `malloc_trim(0)` after each file so freed pages return to the OS instead of ratcheting RSS up to the worst single file. ~6% peak RSS reduction measured on `jac check jac/jac0core/`, with byte-identical diagnostics. The trim is a no-op where `malloc_trim` is unavailable (musl, macOS).
