- **Fix: Blocking IO in Sync Walkers/Functions Stalling the Async Event Loop**: Walker and function endpoints no longer serialize concurrent requests when user code contains blocking IO (`time.sleep`, `requests`, byllm LLM calls, heavy computation). `ExecutionManager` now exposes explicit `spawn_walker_sync` / `spawn_walker_async` and `execute_function_sync` / `execute_function_async` paths; `ExecutionHandler` dispatches at the handler seam based on walker `__jac_async__` / function coroutine kind. Sync user code runs via `asyncio.to_thread` so the event loop stays responsive; async user code is awaited directly. The unified `spawn_walker` / `execute_function` methods are preserved as thin dispatchers for backwards compatibility. The `Memory` interface also gains default async mirrors (`aget`, `aput`, `adelete`, `ahas`, `acommit`) that every backend inherits via `asyncio.to_thread`, enabling future native-async backends (aiosqlite, asyncpg, motor) to override and drop the thread hop without touching callers.
