- **Feature: bundled native `secrets` over a libcrypto CSPRNG floor (#6978 Phase 1)**: `import secrets` now resolves on the native (`nacompile`/JIT) pathway to a bundled `na_stdlib` Mechanism-F module, a thin pure-Jac surface (`secrets.na.jac`) over an OpenSSL `RAND_bytes` FFI floor (`_csprng_native.na.jac`) on the same bundled `libcrypto.a` the `hashlib`/`hmac` modules link. `RAND_bytes` is seeded from the OS CSPRNG, the same source CPython's `secrets` draws from via `os.urandom`. Coverage is `token_bytes`, `token_hex`, `token_urlsafe` (base64-url, padding stripped, exactly as CPython renders), and `randbelow` (uniform via rejection sampling, matching CPython's `SystemRandom._randbelow`); a `RAND_bytes` failure raises `ValueError` rather than returning the zero buffer, and an `nacompile`d binary statically links `libcrypto.a` and runs self-contained. A new tri-backend equivalence test (`prim_secrets`) pins the native and CPython implementations congruent on the deterministic properties of the random output (token lengths including the empty case, `randbelow(1) == 0`, the in-range bound, and that two draws differ).
