- **Project kinds are now a type**: a project kind's facts (entry file, description, action, verb, flags, summary) were split between a stringly-typed `KIND_SPECS` table and a long per-kind dispatch ladder, with entry filenames and descriptions duplicated again in the template registry. A kind is now described by a single typed `KindSpec`, planning a run is a table lookup rather than a dispatch chain, and the vestigial `plugin` field — empty for every kind since plugins were folded into core — has been removed along with the dead gates it fed.
