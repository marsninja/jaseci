- **Feature: Decouple the native (`na`) backend from the Python `llvmlite` package**: the LLVM IR builder and the ctypes binding over the `LLVMPY_*` C ABI are now first-party Jac code under `jaclang/compiler/passes/native/llvm/` (py2jac-translated from llvmlite 0.47.0, BSD-2), and the verbatim C++ `LLVMPY_*` shim lives at `jac/native/`. A new `zig build jacllvm -Dllvm-dir=...` step compiles the shim and statically links a host LLVM 20.1.x into `libjacllvm.so` (the in-tree replacement for the bundled 167 MB `libllvmlite.so`). No `import llvmlite` remains in the runtime; a guard test enforces it. See #6925.
