- **Feature: bundled native `socket` (TCP client) over a libc BSD-sockets floor (#6978 Phase 1)**: `import socket` now resolves on the native (`nacompile`/JIT) pathway to a bundled `na_stdlib` module -- a pure-Jac client surface (`socket.na.jac`) over a thin libc FFI floor (`_socket_native.na.jac`). Unlike the `hashlib`/`secrets` floors that statically link a bundled `libcrypto.a`, the socket primitives are plain BSD sockets in libc (`socket`/`connect`/`send`/`recv`/`getaddrinfo`), so a socket program links nothing beyond libc/libm and is self-contained with no bundled archive -- and, critically, with **no libcurl dependency** (the foundation for moving `http`/`urllib` off the libcurl floor). DNS resolution walks `getaddrinfo`'s `struct addrinfo` list with no new compiler intrinsic: a libc `memcpy` whose source pointer is carried as a plain Jac `int` (the established `FILE*`-as-`int` FFI convention) copies the sockaddr out and `int.from_bytes` reads the fields. Coverage is a blocking TCP **client**: the `socket(family, type, proto)` object with `connect`/`send`/`sendall`/`recv`/`close`/`fileno`, the family-spanning `create_connection((host, port))` helper that the `ssl` wrapper and `http`/`urllib` build on, and the `AF_*`/`SOCK_*` constants. Setup failures and transport errors raise `OSError`, mirroring CPython (whose `socket.error` *is* `OSError`); the raise unwinds cleanly to a caller's `except`, including across a module boundary (relying on the native cross-module exception fix, #7021). A new equivalence suite (`test_socket_equivalence`) stands up a loopback HTTP server and pins the native client congruent with CPython's `socket` on the HTTP status and the exact total response byte count of a hand-driven `create_connection`/`sendall`/`recv`-drain GET, and asserts that connecting to a closed port raises a catchable `OSError` (not an abort). Server-side (`bind`/`listen`/`accept`), UDP, `settimeout`, and `makefile` are documented follow-ups; the Linux/glibc `struct addrinfo` offsets make this native-host-Linux-first, with macOS/Windows on the cross-platform follow-up.
