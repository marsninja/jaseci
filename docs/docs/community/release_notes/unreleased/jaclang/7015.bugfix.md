- **Fix: ICE when `root`/`super` name a parameter**: Using the reserved refs `root` or `super` as a parameter name (in a `def`/`can` signature or a `lambda`) no longer crashes the compiler with an Internal Compiler Error in `PyastGenPass` ("Empty py_ast on ParamVar") or a bogus `W2003` "defined but never used". Because they lower to calls (`Jac.root()`/`super()`) they have no valid parameter form, so the parser now reports `E0013` ("keyword cannot be used as a parameter name") like any other keyword. Escape with a backtick (`` `root ``) to bind a local of that name. This also clears the precompile failure on the in-tree `jir.jac` (`scrub_cache_dirs`).
