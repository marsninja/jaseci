- **Fix: five native memory-management bugs found by zero-RC vs RC differential benchmarking (#7466)**: per-function `param_vars` state leaked across function codegen, so any function whose param names shadowed another function's locals silently perturbed RC codegen — a fresh object returned under a shadowed name gained a spurious retain-on-return (the caller then leaked every returned value, under `rc` and `cycles` alike), and loop-exit/scope-exit cleanup skipped shadowed loop-body locals (leaking the final iteration's values, headerless `--gc none` and `rc` alike); heap-element containers (`import sys`'s argv, or a plain `list[str]`) crashed codegen under `--gc rc`/plain `--gc none` with `DuplicatedNameError: __rc_debug` because the element-release emitter re-entered `_emit_rc_helpers` through a cycles-only guard — the helper emitter is now idempotent and the mid-collection check is emitted only under `cycles`; `def drop` hooks now fire under managed gc modes too, invoked from the reference-count destructor at the same last-use point headerless drop insertion uses, restoring differential-identical output across gc modes (timing and caveats now documented in `ownership-borrowing.md`); and `--enforce-nogc` no longer move-tracks scalar call-result locals, fixing the bogus `E1301` on `iters = pow2(m); for i in range(iters)`.
