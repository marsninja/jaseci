- **Serve: route-aware SPA fallback returns honest 404s**: the catch-all that serves the client app for clean URLs now consults the route table derived from the project's `pages/` directory, so paths no client route can render get the app shell with status 404 instead of a soft-404 200. This fixes SEO/monitoring semantics and stops stale long-poll clients (for example an mkdocs livereload tab left pointing at the port) from busy-looping at hundreds of requests per second, since such clients only back off on non-200. Dynamic segments, nested catch-alls, and route groups still serve 200; projects without a `pages/` directory (manual Router apps) keep the permissive fallback.
