- **Perf: Cache filter introspection by lambda `__code__`**: `_filter_type_name` and `_filter_has_predicates` are now memoized via `functools.lru_cache` keyed on `filt.__code__` (through a `_CodeKey` carrier where the closure walk is still needed). Jac emits filter lambdas inline at the path-construction call site, so identity-keyed caches never hit; code-keyed caches collapse every fresh function-instance from the same source to one entry. On the LittleX5 `load_feed` profile this cut filter introspection from ~2.65s / 43% of `osp_spawn` per walker run to a single-digit number of `dis.get_instructions` and `inspect.getclosurevars` invocations across the whole benchmark.
