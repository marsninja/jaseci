- **Refactor: one type-based route resolver for the client file-based router**: route membership (introduced as the `JsxPage` / `JsxLayout` return types in #7654) is now resolved through a single shared `resolve_route_exports` used by both the client build and the server's SPA fallback. The resolver-less filename fallback in `RouteScanner` is removed (`LAYOUT_FILENAME`, `LEGACY_PLACEMENT_SUFFIXES`, and the `.cl.`/`layout.jac` heuristics go with it); `route_exports_of` is now required. The server's honest-404 SPA fallback enumerates routes from source via the same resolver, so it no longer over-counts a co-located `JsxElement` component as a route. No behavior change for real routes; net -29 lines.
