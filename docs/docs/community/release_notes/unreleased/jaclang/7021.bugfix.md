- **Fix: cross-module exception unwinding in the native backend**: A `raise` in one `.na.jac` module caught by a `try`/`except` in a different module aborted the process (SIGABRT) instead of unwinding to the handler. The setjmp/longjmp handler-stack head (`@__jac_current_handler`) is now a `weak` link-merged singleton shared across all linked modules, and exception type IDs are now stable and module-independent (derived from the canonical builtin hierarchy), so a cross-module raise reaches and matches its handler.
