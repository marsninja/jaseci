- **Breaking: connect operator returns the RHS as-is (#6817)**: The connect operators (`++>`, `<++`, `<++>`, and the typed `+>:Edge:+>` form) no longer always wrap their result in a list. A connect expression now mirrors what it connects to -- connecting to a single node returns that node, and connecting to a list of nodes returns a list. The `(a ++> b)[0]` idiom used to unwrap a single connected node is no longer needed (and now errors); drop the trailing `[0]`. List connects and chaining (`a ++> b ++> c`) are unaffected.
