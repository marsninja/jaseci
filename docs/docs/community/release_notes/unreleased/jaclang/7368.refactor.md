- **Perf: Gate the native LLVM backend for non-native compiles**: `NaIRGenPass` and `NativeCompilePass` are only scheduled when a module actually needs native codegen (a `.na` file, auto-promote, or any native-context statement -- inline `na {}` blocks and native-prefixed decls). A non-native process (a plain Python/client compile, or a `jac start` server for a non-native app) now imports zero native pass modules and never maps the `libjacllvm` shared library, versus 29 modules plus the dylib before. On a clean cache this trims about 26 MB RSS per process on macOS, where the ~91 MB `libjacllvm.dylib` is demand-paged; the saving is larger on Linux, whose `libjacllvm.so` is ~130 MB. Mixed sv/na modules still JIT into the native engine so live sv->na calls resolve. The scale deploy/observability factories are also imported lazily inside the `deploy_*` commands, so a local `jac start`/`jac serve` dev server no longer compiles those subtrees.
