- **Editable dev loop now active when running from `jac/`**: The `[dev] jaclang_source` dev loop is read from the nearest `jac.toml`, so `cd jac && jac test tests` previously fell back to the binary's bundled jaclang (and ignored your source edits) because `jac/jac.toml` carried no `[dev]` stanza. `jac/jac.toml` now declares it alongside the repo root, so the loop holds from the jaclang dir. Set `JAC_NO_DEV_SOURCE=1` to force the loop off for a single command (e.g. to test the shipped bundled + precompiled jaclang).
