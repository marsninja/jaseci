- **Fix: native pathway crash family -- slices, negative repeat, count(""), exception subclass fields, varargs**: five S2 crash/corruption shapes from jaseci-labs/jaseci#7320 Cluster 3. String and list slices now follow CPython semantics (negative indices wrap, bounds clamp, steps walk either direction, zero step raises ValueError) instead of bus-erroring on `s[1:-1]` or returning empty/unreversed results. `"x" * n` with negative n yields `""` instead of corrupting the heap. `s.count("")` returns len+1 instead of hanging forever. Raising a user-defined exception archetype now carries the real object through the unwind so `except ParseError as e { e.line_no }` reads the raised fields instead of garbage. And `*args`/`**kwargs` defs are rejected with a loud E5090 instead of miscompiling into a runtime segfault.
