- **Fix: the `deploy.k8s` capability closes over `python-dotenv`**: the deploy path reads the project `.env` through `dotenv` in the Kubernetes realizer and in the manifest builder, but `python-dotenv` was declared only in the `serve` and `env` capabilities. On a clean operator machine `jac start --scale` crashed with `No module named 'dotenv'` before jac's own deploy-tooling auto-install could run, and project `[dependencies]` could not help because the deploy path resolves from the jac global site. `python-dotenv` is now pinned in `deploy.k8s`, and the capability-closure test scans the deploy tree's imports so the pin set can no longer drift from what the code uses.
