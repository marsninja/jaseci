- **Fix: native acceleration no longer clobbers imported foreign functions**: after a fresh compile of a module with an `na{}` region, `install_native_wrappers` replaced every module-namespace name that also appeared in the native layout, including the exported functions of linked native imports. A module holding CPython's `loads`/`compress`/`wrap` under the same top-level name (`import from json { loads }`) had its Python binding silently swapped for the native implementation, producing wrong values, kwarg/arity `TypeError`s, or ctypes-level crashes on the Python side. Wrappers are now installed only over bindings the module itself defines (matching `__module__`); the module's own functions accelerate as before. The nine bundled-prim equivalence tests this affected (zlib/gzip/bz2/json/base64/textwrap/csv/pprint/difflib) are re-enabled.
