- **Fix: the microservice-mode banner no longer tells you to set `[scale.microservices]` when it already is**: `jac start --scale` printed "default routes ... set [scale.microservices] to customize" and the production-readiness warning on every deploy, even when the section was set and honored, so a genuinely degraded config was indistinguishable from a healthy run. The banner (kubernetes and local serve alike) now reports the routes actually applied, split into `configured:` and `auto-derived:` lines, and only shows the customize hint and warning when jac.toml genuinely lacks a `[scale.microservices]` section.
