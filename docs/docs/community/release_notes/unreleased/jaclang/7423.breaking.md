- **Breaking: `jac create --list_jacpacks` is now `jac create --list`**: the flag never listed jacpacks. A `.jacpack` is a distributable bundle produced by `jac create --pack <dir>` and consumed by `jac create --use <path|url>`; the flag instead lists the project *kinds* (used with `--kind`) and *named variants* (used with `--use <name>`) in the template registry, as its own help text already said. The misleading name also forced an awkward spelling, since `--list-jacpacks` was rejected and only the underscored `--list_jacpacks` parsed. This is a clean break with no deprecated alias: `--list_jacpacks` now fails with `unrecognized arguments`. Replace it with `--list`; the short form `-l` is unchanged, and nothing about the `.jacpack` format, `--pack`, or `--use` changes.
