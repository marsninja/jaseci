- **Feature: native `zlib` standard library over an FFI floor (#6940 Phase 2)**: `import from zlib { compress, decompress, crc32, adler32 }` now resolves to a bundled `na_stdlib/zlib.na.jac` on the native (`na`) pathway (and binds CPython's `zlib` on the sv pathway, the same source on both). It follows the FFI-floor + pure-Jac-surface architecture: the DEFLATE engine is the system `libz` reached through a thin FFI binding (`_zlib_native.na.jac`, Mechanism F, libz bound by logical name `z`), with the Python-shaped API (CPython argument orders and defaults, zlib-format streams, byte-for-byte congruent compressed output, grow-on-demand `decompress`) layered on top. To make foreign byte I/O work, a `bytes`/`bytearray` argument to a clib function now lowers to its raw data pointer (the C buffer-protocol convention) rather than the internal jacbytes `{ i64 len, [n x i8] }` struct pointer, so C no longer reads the 8-byte length header as leading data. A tri-backend equivalence fixture pins sv/na congruence on the checksums and the compress/decompress round-trip.
