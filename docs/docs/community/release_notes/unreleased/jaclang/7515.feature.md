- **Gateway-owned identity for microservice mode**: the gateway now serves `/user/register`, `/user/login`, `/user/refresh-token`, `/user/password`, and `/user/me` from its own user manager instead of passing them through to an arbitrary service, and services JIT-provision a local user record + root for validly-signed tokens they have not seen (microservice siblings only). Previously, identity landed in whichever service answered the passthrough first, so walkers in any other service failed with a 500 `User not found`; that failure also now maps to 401 instead of 500.
- **Fix: stale client bundles in microservice mode**: `jac start` skipped the client build whenever `dist/index.html` existed; it now rebuilds when any project source is newer than the built bundle. The build timeout also rises from 120s to 600s - the old ceiling killed legitimate cold builds after their success banner had printed.
- **Fix: `jac scale status` sees sibling-spawned services**: running-state came from the current process's in-memory table, so a status run in another terminal showed live services as "(not running)". Service ports now persist beside pidfiles (`.jac/run/<name>.port`) and status recovers pid/port from disk, gated on the health probe answering.
- **Fix: `jac browse` usability**: missing positional args produce `usage:` errors instead of a leaked `list index out of range`, and the element-locate stability window widens 50ms->150ms so scroll-triggered entrance animations cannot slip a click to a stale position.
- **Fix: quieter client builds**: the per-module "Builtins module is already bound, skipping." INFO line (~50 copies per bundle build) drops to debug level.
