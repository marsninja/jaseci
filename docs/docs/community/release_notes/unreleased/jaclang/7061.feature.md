- **Feature: bundled native `fractions` (#6978 Phase 2)**: `import from fractions { Fraction }` now resolves on the native (`nacompile`/JIT) pathway to a bundled pure-Jac `Fraction` (`na_stdlib/fractions.na.jac`), GCD-normalized to CPython's value model. Construction and reduction (`Fraction(3, 6)` -> `1/2`, sign normalization, whole numbers, zero), the `numerator` / `denominator` properties, and `str()` match CPython's `fractions.Fraction`. Arithmetic and ordering are the CPython dunder methods (`__add__`, `__sub__`, `__mul__`, `__truediv__`, `__eq__`, `__lt__`), called directly because the native backend has no operator-overload dispatch yet. Float/Decimal/string construction is out of scope. Pinned sv<->na congruent by the new `prim_fractions` equivalence fixture.
