- **Refactor: native RC-elision is now proven by the backend, not the ownership checker (#7149)**: The reference-count elision for move assignments (`b = a`) previously depended on `Symbol.is_proven_unique`, a fact stamped by `OwnershipCheckPass` -- which runs only in the type-check schedule. Because `nacompile` compiles with `type_check=False`, that fact was absent on the native path, so the backend could act on stale or missing ownership metadata (a use-after-free hazard) or silently elide nothing. The ownership checker is now strictly diagnostics-only: it writes no backend-consumed fact, and `Symbol.is_proven_unique` is removed. A new `RcElisionProofPass` runs unconditionally in the native codegen path (before `NaIRGenPass`) and proves elision itself from the syntactic AST and the CFG -- never from checker or symbol state -- so native RC elision now works identically with `type_check=True` and `type_check=False`. The proof is conservative and scoped to the exact move site (`Assignment.na_move_lowerable`): it elides only a plain local source whose every use is an alias-value into a plain local and which backward liveness proves dead-out at the move, which makes the previous unsound channels (own-parameter elision, escaping/borrowed sources, and loop-carried re-reads across a back-edge) unrepresentable rather than merely unflagged.
- **Refactor: region-escape analysis is now built on an access-path (`Place`) abstraction with a default-deny posture (#7149)**: The arena escape checker previously carried three separate copies of the same `UnaryExpr → AtomTrailer → Name` unwrap walk (`_region_root_name`, the container-literal recursion in `_region_escapees`, and `_return_roots_at_param_borrow`) and could only see through borrow and container-literal chains -- so a region-local routed through a function call, comprehension, ternary, or f-string in an escaping position silently passed as a use-after-free once the arena freed. Those walks collapse into one `Place` constructor (a root symbol plus an ordered projection list) and one `_places_reachable` collector with a default-deny stance: a call is assumed to alias any argument or its receiver, and any construct that cannot be classified as copying falls to a conservative net, so a missed escape (a real UAF) is now surfaced as `E1307` rather than passing silently. Concurrent (`flow`/`wait`) payloads route through the same collector instead of a blunt "grab every name" scan. The `Place` projection list also positions later field-granular borrow diagnostics (disjoint field borrows, move-out-of-field) to reuse the same representation rather than adding a fourth walk.
- **Refactor: `&x` is defined as erased on the Python backend, and the checker's edges are pinned down (#7149)**: A prefix `&x` used to lower to the archetype-lookup call `jobj(id=x)` on the Python backend; the borrow-checker work repurposed `&`/`&mut` as borrow expressions, which made that branch unreachable. The dead lowering is now removed and the semantics are documented: on every backend, ownership annotations are compile-time-only, and `&x`/`&mut x` compile to exactly `x` (call `jobj(id=...)` explicitly for an id lookup). Alongside this, `__rc_collect_cycles` -- a native-only test hook with no Python runtime implementation -- is no longer whitelisted as a Python-backend builtin (calling it there now surfaces an unresolved-name diagnostic at check time instead of silently type-checking and failing with a runtime `NameError`); the native backend resolves the `__rc_*` intrinsic family by name ahead of builtin classification, so the hooks keep working there without the registry entry, the JIR format version is bumped to 14 for the `OwnershipKind` field wiring, and a new spec document (`docs/reference/language/ownership-checker-spec.md`) states what each `E13xx` code guarantees, the checker's symbol-level opt-in granularity, and the diagnostics-only contract that forbids any backend from consuming checker output. Finally, the cycle-collector work grew the native object header from 32 to 48 bytes (adding a trace-function slot and a cycle-collector color slot), which shifts every header field offset -- a hard native-ABI break, so all native artifacts must be recompiled together (there is no cross-version header compatibility).
