- **Breaking: hermetic, self-contained installs**: `jac` now works regardless of the surrounding environment (with or without a host Python), and third-party packages live only where you put them. `jac install <pkg>` and `jac install -e <path>` install into the current project's `.jac/venv` by default (and require a `jac.toml`); pass `--global` to install into the binary's own jac-owned site instead, for a plugin or tool you want importable from any project. Installs always run through the binary's bundled pip (no external `uv`); `.jac` module resolution and plugin discovery no longer consult the host's system/user `site-packages`; and an activated outside venv on `PATH` can no longer shift the interpreter's `sys.prefix`. `jac bundle --precompile` now precompiles for the bundled interpreter only (consumers on another Python recompile on first import).
