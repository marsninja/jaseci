- **Perf: bound compile-time memory by releasing per-unit AST closures**: long-lived compile processes (boot, precompile, the language server) no longer accumulate every module's AST closure. Each module's working set — hub, type-evaluator caches, dependency map, and diagnostics — is released at its end-of-unit boundary, with the `.jir` cache as the durable cross-unit channel. A full cold precompile of `jaclang` drops peak memory ~9x (≈8.5 GB → ≈1.0 GB). Set `JAC_NO_UNIT_EVICT=1` to opt out.
