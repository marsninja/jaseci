- **Feature: native programs can static-link a bundled C floor into ELF binaries (#6978 Phase 0.1a)**: extends the static C-archive linking from #6986 (Mach-O) to ELF and wires it into `jac nacompile`, so a clib floor import like `import from lzma { def lzma_crc32(...) }` links the bundled archive into a Linux binary instead of carrying a runtime `.so` dependency. Adds `nacompile --target linux` to cross-build ELF from any host. Two ELF-linker fixes were needed: a section-relative relocation from a merged archive member now resolves to the member's offset within the coalesced section (not the section base), and the injected `main` runs the merged `.init_array` constructors before entry (the bundled `_start` bypasses libc startup, so a floor member's load-time constructor would otherwise never run). Verified in Linux containers on x86_64 (constructor and multi-member floors) and arm64 (constructor-free floors); an arm64 ADRP/ADD relocation edge for constructor-heavy members remains a follow-up. macOS Mach-O is unaffected.
