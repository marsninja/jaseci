- **Feature: bundled native `gzip` over the `zlib` floor (#6978 Phase 2)**: `import from gzip { compress, decompress }` resolves on the native (`nacompile`/JIT) pathway to a bundled `na_stdlib` module (`gzip.na.jac`) -- a pure-Jac (Mechanism B) surface that adds the RFC 1952 gzip header / CRC-32 / ISIZE framing around the raw DEFLATE body of the existing `zlib` floor, with **no new FFI**: gzip is just zlib's DEFLATE engine plus a 10-byte header and an 8-byte trailer, and the `zlib` floor's one-shot `compress2` / `uncompress2` already provide the engine. `compress(data, compresslevel=9, mtime=0)` strips the 2-byte zlib header and 4-byte adler32 trailer off a zlib stream to recover the raw DEFLATE body (identical bytes under either frame) and wraps it; the output is byte-identical to CPython's `gzip.compress` at the same level and `mtime` (XFL 2 for level 9, 4 for level < 2, 0 otherwise; OS byte 255; CPython 3.14 also defaults `mtime` to 0, so the defaults agree byte-for-byte). `decompress(data)` walks the members of the stream exactly as CPython does: per member it parses the header (honoring the FEXTRA / FNAME / FCOMMENT skips; FHCRC bytes are skipped unverified, matching CPython), re-frames the remaining input as a zlib stream so the member's own trailer bytes stand in for the adler32, inflates it through `uncompress2` -- whose consumed-source count locates the member boundary -- and enforces gzip's own CRC-32 + ISIZE (mod 2^32, per RFC 1952) before concatenating the member outputs; truncated members and trailing garbage raise, as in CPython. Error types map `gzip.BadGzipFile` / `EOFError` / `zlib.error` to `ValueError` on the native surface; the `GzipFile` class and streaming file API are out of scope. The equivalence fixture (`prim_gzip`, registered with `require=["na"]`) pins the native module congruent with CPython's `gzip` on compressed-stream byte-identity (poly digest + exact length + XFL byte at levels 0/1/6/9 and the default, `mtime=0`), round-trip (including a level-0 stored-block stream), multi-member concatenation, empty input, hand-spliced FEXTRA / FNAME / FCOMMENT / FHCRC headers, and seven raising error paths compared as flags.
