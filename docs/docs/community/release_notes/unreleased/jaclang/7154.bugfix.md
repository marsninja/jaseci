- **Fix: `.na.jac` / `.sv.jac` test files silently collected 0 items (#7150)**: The pytest collector derived a test module's import name with `Path(...).stem`, which left the compound codespace suffix in place (`test_foo.na.jac` -> `test_foo.na`), so the importer read the trailing `.na` as a package path, failed, and the failure was swallowed into an empty result -- `jac test test_foo.na.jac` reported "no tests ran" and exited 0, letting CI go green while executing nothing. Module names are now derived through the extension registry's canonical `base_stem` (the single source of truth for suffix stripping), so `foo.na.jac` / `foo.sv.jac` import as module `foo` and their `test` blocks collect and run. Additionally, a test file that was collected but fails to import is now surfaced as a collection error (matching pytest's contract for `.py` test files) instead of being silently swallowed into a passing run.
