- **Fix: stdlib re-parsed from source on every `jac` invocation (cold start)**: the payload builder now precompiles the bundled CPython stdlib and the jaclang site to hash-based (`unchecked-hash`, PEP 552) `.pyc` instead of relying on pbs's shipped timestamp `.pyc`. The runtime materializer untars the payload into a fresh cache dir and `std.tar` carries no mtime, so every extracted `.py` gets a "now" timestamp that can never match a timestamp-`.pyc`'s embedded build-time mtime; combined with the embedded interpreter running `write_bytecode=0`, the interpreter discarded the `.pyc` and could not re-cache one, re-parsing every hot stdlib module (`asyncio`, `logging`, `http.server`, `pdb`, ...) from source on every run. Hash-based `.pyc` ignore mtime and survive materialize, cutting `jac --version` from ~888 ms to ~250 ms (warm, shipped path). The step fails safe: any module that cannot compile just recompiles at runtime as before.
