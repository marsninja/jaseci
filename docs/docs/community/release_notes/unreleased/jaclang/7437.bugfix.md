- **Fix: `--scale` deploys reach single-node local clusters without an RWX bundle PVC**: the source-bundle store unconditionally required a `ReadWriteMany` PVC plus an explicit `bundle_storage_class`, so kind / k3d / minikube clusters (which ship no RWX provisioner) could not be targeted without hand-provisioning static hostPath PVs, even though the docs referenced a hostPath escape hatch that did not exist. A new `bundle_host_path` key under `[scale.kubernetes]` now stages bundles in a directory on the host node instead of a PVC (on a single node RWX degenerates to a shared hostPath), is mutually exclusive with `bundle_storage_class`, and is named by the fail-fast error raised when neither bundle store is configured.
