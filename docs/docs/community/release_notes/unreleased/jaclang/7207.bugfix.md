- **Fix: Installed-but-untyped third-party imports no longer hard-error**: Attributes and calls reached through an installed dependency that ships no type information (no PEP 561 `py.typed` marker and no `.pyi` stub) inferred as `Unknown` and the strict-`Unknown` discipline then errored on every downstream access (E1030/E1032) and assignment (E1002), keeping real `requests`/`streamlit`/`kubernetes`-style apps uncheckable. Such modules now degrade to a permissive foreign `Any` at the import boundary (surfaced once via the configurable `untyped-external` diagnostic), while strict `Unknown` stays reserved for the project's own source and packages shipping `py.typed` remain fully analyzed.
