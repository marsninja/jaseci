- **Native stdlib (#6940 Phase 1): bundled `json` + `os.makedirs`/`os.path.realpath`**: A new bundled `na_stdlib/json.na.jac` ships a pure-Jac (Mechanism B) `json.loads` (recursive-descent parser over boxed `any`: dict/list/str/int/float/bool/None, with full string escapes and int-vs-float parsing) and `json.dumps` (CPython-default `(', ', ': ')` separators, insertion-ordered keys), so the same `import from json { loads, dumps }` binds CPython `json` on the sv pathway and the native module on na. The `os` intercepts gain `makedirs` (a `mkdir -p` loop over libc `mkdir`) and `os.path.realpath` (libc `realpath(3)`, non-strict). A supporting native-backend fix gives a boxed `None` its own JacVal tag instead of masquerading as a null string (which made `isinstance(None, str)` true and crashed `str`/`len`/iteration on it), so `json.dumps(None)` and null object values work. Known divergences: `dumps` of a bare float uses `str(float)` (`%g`) rather than the shortest-round-trip repr (gated on the Ryu float repr, Phase 0.3), and string escaping is ASCII-congruent (`ensure_ascii` of non-ASCII is a follow-up).
