- **Fix: the `deploy.k8s` capability closes over `bcrypt`**: the capability registry pinned only `kubernetes` and `docker`, but the Kubernetes ingress deployer needs `bcrypt` to hash the RedisInsight basic-auth secret. Because deploy tooling is auto-installed from this dep set, a scale deploy would sync its own tooling and then fail with `No module named 'bcrypt'`. The dep set is now complete.
- **Fix: `jac start --scale --dry-run` help text describes what the flag actually does**: it claimed to "touch nothing (no cluster contact, no build, no DB)" while installing deploy tooling into the jac global site. The install is gone (see jac-scale), and the text now states the guarantee it can keep -- nothing applied to the cluster, no DB provisioning, no TLS issuance, no deploy tooling installed.
