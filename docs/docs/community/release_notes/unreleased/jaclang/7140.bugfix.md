- **Bugfix: capability closure for scale serves no longer depends on intent truthiness** (issue #7137): `derive_capabilities` defaults the `data.mongo` user store for a scale serve based on the scale/serve context (or any declared intent), so a project with no `[plugins.scale.*]` section still resolves pymongo into its closure, while plain local serves stay lean. The capability registry also gains `missing_capability_dists` and `ensure_global_capability`, which sync a capability's pinned dependencies into the jac-owned global site (the same site `jac install --global` targets) so command-context tooling like `deploy.k8s` can install itself on first use.
