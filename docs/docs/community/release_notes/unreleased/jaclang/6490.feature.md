- **Feature: Solid (signals) client backend**: `framework = "solid"` now selects `SolidBackend`, the first fine-grained reactivity target for client codegen (`.cl.jac` / `cl {}` → JavaScript). Unlike the React/Preact re-render model, Solid runs each component body once and wires a reactive graph, so the backend exercises the read seams added in the prior phases: reactive state declarations lower to `createSignal`, value reads become accessor calls (`count()`) via `lower_state_read`, and dynamic JSX children/attributes become thunks (`() => count()`) via `lower_view_expr` so the runtime can re-track each binding. Writes route through the setter (`setCount(...)`); augmented assigns use a functional updater (`count += 1` → `setCount(prev => prev + 1)`) to avoid stale reads and self-subscription. Effects lower to `createEffect` (entry) and `onCleanup` (exit); the shared `__jacJsx` factory targets Solid's hyperscript runtime (`solid-js/h`), and `try/awaiting/except` slots lower to `<Suspense>`/`<ErrorBoundary>`. React/Preact output is unchanged (every new lowering is a no-op on the hooks backends). v1 is a best-effort subset over the representative fixtures; documented gaps (per the reactivity contract) include deferred `Ref[T]` fields (lowered to a plain `let` binding, no `ref={...}` wiring yet), block-level shadowing of reactive fields, the functional-updater asymmetry for plain self-referential assigns, true `createResource`-backed suspension, and full `dom-expressions` parity.
- **Feature: Solid runtime router + form parity**: the Solid runtime shim (`solid_runtime.cl.jac`) now exposes the same router, form, and error/async surface as the React shim, so `@jac/runtime` stays framework-identical for app code while a Solid build still pulls in **no** React-ecosystem package. Routing is backed by `@solidjs/router` through thin adapters that bridge the API-shape gap (`Link` → `A`, `to` → `href`, `Route element=` → `component=`; `Routes`/`Outlet` are best-effort shims), covering `Router`/`Routes`/`Route`/`Link`/`Navigate`/`Outlet`/`useNavigate`/`useLocation`/`useParams` plus the composed `useRouter` and `AuthGuard`. Forms use `@tanstack/solid-form` (`createForm`) with Zod schemas passed directly as Standard Schema validators (no resolver), so `useJacForm`/`JacForm`/`JacSchema` work without any `@hookform/*` package; per-field error messages stay reactive because the schema-driven `JacForm` reads `field.state.meta.errors`. (The form layer was later migrated to TanStack Form on both React and Solid in #6539; see that note for the current state.) Error/async parity (`JacClientErrorBoundary`/`ErrorFallback`/`JacAwaiting`) is built on `solid-js`'s `ErrorBoundary`/`Suspense`. A Solid project's default client dependencies are now framework-aware (`solid-js`, `@solidjs/router`, `@tanstack/solid-form`, `zod`; no React packages). File-based routing entry codegen is backend-aware (`_create_pages_entry_content` delegates to `build_pages_entry_script` for React and Solid). The simple Solid mount entry now wraps `app` in `JacClientErrorBoundary`, matching React/Preact. Runtime npm publishing rejects unknown peer packages instead of emitting an unbounded `"*"` range. Documented gaps pending browser e2e: nested-route matching through the `Routes` passthrough and true `Outlet` semantics on `@solidjs/router`.
- **Feature: View IR seam (phase 2)**: JSX lowering now builds a framework-neutral `ViewElement` tree in `EsJsxProcessor` (tag/attribute/child classification happens once) and hands it to `FrameworkBackend.lower_view`. React/Preact/Solid share `ReactBackend.lower_view`, which rebuilds the same `__jacJsx(tag, props, [children])` call; dynamic positions are thunked inside `lower_view` via `lower_view_expr` (no-op on hooks backends). Attribute kinds (`StaticAttr`/`DynamicAttr`/`EventAttr`/`RefAttr`) are classified in the producer so event handlers and `ref` never pass through the view thunk seam. React/Preact output is unchanged (zero-diff gate preserved).
- **Feature: native Solid file-based routing (phase 2)**: the generated Solid pages entry now targets `@solidjs/router` natively: `<Route>` children sit directly under `<Router>` (no `<Routes>`), routes use `component={Page}`, protected routes nest under a pathless `AuthGuard` route that renders `props.children`, and root layouts ride `<Router root={__jacMakeLayoutRoot(Layout)}>`. `Outlet` bridges Jac layout authoring (`<Outlet/>` in a param-less layout) to Solid Router's `props.children` via `__JacOutletContext`. jsdom routing fixtures exercise flat, nested, auth-guard, and layout routes.
