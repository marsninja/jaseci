- **Refactor: Impl/test annexes and variants merged into the unified module tree**: `.impl.jac`/`.test.jac` annex bodies and `.sv.jac`/`.cl.jac`/`.na.jac` variant bodies are now spliced into their head module's own tree by a single pass (each spliced statement keeping its own deployment context and file provenance), instead of hanging off `Module.impl_mod`/`test_mod`/`variant_mod` side-lists that the generic walk, sub-node table, and JIR serializer could not see. Every compiler walk, the JIR cache, and tooling now cover annex and variant content directly; the side-list fields and the recompile-on-cache-hit reattachment hack are removed. Cross-variant `impl` matching (a head `.impl.jac` implementing a `.cl.jac`/`.sv.jac` declaration) falls out of the single same-module decl/impl connect.
- **Fix: Auto-lint signature fixing across separate `.impl.jac` files**: `jac format --fix` repairs an impl's parameter list to match its declaration even when the declaration lives in the base file; the auto-lint pass loads the base module's declarations on demand now that each annex is formatted as its own tool program.
- **Fix: Duplicate diagnostics inside matched `impl` bodies**: A type error in a separated `impl` body (e.g. `def f; impl f { ... }`) was reported twice. It is now reported once.
- **Fix: Annex semantic tokens in the language server**: Hovering, go-to-definition, and semantic highlighting inside an `.impl.jac`/`.test.jac` file now resolve again, served per source file by provenance from the merged tree.
