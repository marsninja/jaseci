- **Type checker: permissive `except` binding for unresolved exception classes**: An `except X as e` clause no longer fabricates false positives when `X` does not resolve to a real class declaration. Previously `e` bound as `Unknown` for an unresolved/untyped import (hard-erroring `E1032` on every attribute access) or as a guessed `Exception` supertype when `X` was a bare Name reassigned in an optional-dependency shim (erroring `E1030` on domain attributes such as `e.status`). Such handlers now degrade gracefully to `Any`, while a handler bound from a genuine class (including an imported one) keeps its type so bad attribute access is still caught.
