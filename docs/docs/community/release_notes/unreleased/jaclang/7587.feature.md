- **Feature: demote-on-lowering-failure completes the codespace preference model; autonative retired**: `[build] default_codespace = "na"` is now a *total* preference, safe on any codebase. A markerless module the closure classifier infers native but that fails native compilation (the E5090/E5092 lowering-coverage class) is transparently recompiled in the server codespace with a dim `note:` instead of erroring, and importers of a demoted module demote with it (a process-level demotion memo keeps re-classification cheap and deterministic). Because the native stdlib is congruent with CPython, demotion is behavior-preserving: programs simply run server until lowering coverage catches up, then migrate to native automatically. Explicit `.na.jac` files and `na` markers remain mandates and keep failing loudly, as does `jac nacompile`. The now-redundant autonative stack is removed: `jac run --autonative`, `[run] autonative`, the `_auto_promote_native` program flags, the `module_native_compat` promotion gate and its `CapabilityCheckPass`, `gen.native_compat`, and the deferred `native_accel` startup hot-swap. Migration: replace `--autonative` / `[run] autonative = true` with `[build] default_codespace = "na"`.
