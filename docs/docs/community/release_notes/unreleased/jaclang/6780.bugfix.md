- **Fix: `jac eject` crashes on real projects**: Eject now skips `_`-prefixed private abilities (matching the runtime server, which never exposes them) instead of aborting on cross-module name collisions, excludes `.na.jac` native-compile targets from the backend, and sanitizes generated request models (widening non-JSON-safe field types to `Any` and unwrapping the `field(factory=...)` mutable-default idiom) so `backend/main.py` imports cleanly.
- **Fix: ejected frontend rendered unstyled**: The generated `vite.config.js` now carries over the project's `[plugins.client.vite]` plugins and `lib_imports` from `jac.toml`, so plugin-driven tooling like Tailwind v4's `@tailwindcss/vite` runs and CSS utilities are emitted (previously the site loaded with no styling).
- **Fix: ejected static assets (`/static/...`) 404'd**: Eject now mirrors project resources under `frontend/public/static/` (including the runtime's `assets/`-fallback form), so client `/static/<path>` references like images, JSON, and WASM resolve in the dev server and built `dist` instead of falling back to `index.html`. The dev server also proxies `/docs` + `/openapi.json` to the backend so the embedded Swagger panel loads.
