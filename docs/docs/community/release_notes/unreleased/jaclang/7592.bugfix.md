- **Fix: sv-to-sv microservice calls work again on a base install**: two stacked bugs broke every locally auto-spawned service. First, the three service spawn sites (scale sibling spawn, orchestrator `ServiceProcessManager`, client dev backend) still passed the removed `--no_client` spelling (renamed to the documented `--no-client` in #7512), so every spawned service exited at argparse before serving and callers saw only `did not become healthy`; the spawn argv now comes from one shared helper (`sv_client.headless_start_args()`), and a contract test captures each spawn site's real argv and parses it with the actual CLI parser so a future flag rename fails in CI instead of production. Second, the scale plugin's httpx-backed transport returned `None` from `sv_service_call`/`sv_walker_call` when httpx was not installed, silently discarding the cross-service call; the core runtime now gates that delegation on `sv_transport_available()` and otherwise keeps the call on its stdlib transport, and the silent `return None` guards are gone. Skill docs and docs-site pages that still taught the `--no_client` spelling were corrected.
