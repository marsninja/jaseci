- **Breaking: `jac/pyproject.toml` removed -- jaclang is the single binary, not a PyPI/editable package.** The clean break is complete: jaclang ships only as the self-contained `jac` binary (Zig launcher + bundled CPython), built with `zig build` from `jac.toml` (no setuptools/pyproject). There is no more `pip install -e ./jac`. CI and the dev `scripts/fresh_env.sh` build the binary (via the new `.github/actions/setup-jac` action) and put `jac` on PATH; plugins install with `jac install -e <plugin>` (deps go into the project venv, jaclang is host-provided by the binary). The binary now bundles `watchdog` (for `jac start --dev`) and `tomlkit` (format-preserving TOML writes), and `jac bundle --precompile` precompiles through the bundled interpreter, so no system Python, pip, or PyPI jaclang is required for any `jac` workflow.
