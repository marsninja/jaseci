- **Feature: native `str(float)` now emits CPython's shortest round-trip repr (#6978 Phase 0.3)**: `str`/`repr`/`print`/f-string/`str.format` of a float on the native (na) backend previously used a lossy `%g` (`str(1.0)` gave "1" on na vs "1.0" on sv). A memoized `__jac_float_repr` helper now finds the smallest precision whose `%.*e` rendering re-parses (`strtod`) to the exact double, then reformats in CPython's `'r'` style (exponential when `decpt <= -4` or `decpt > 16`, else fixed `%.*f` with a trailing `.0` for integral values; nan/inf/-inf spelled as CPython does). This closes the `json.dumps` bare-float parity gap, and a new equivalence test pins na/sv byte-equality across 33 doubles including subnormals, the max double, `-0.0`, and inf/nan.
