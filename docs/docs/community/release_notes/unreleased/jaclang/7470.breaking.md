- **Breaking: the deep-immutable marker is now the reserved keyword `imm` (was contextual `val`)**: the ownership marker that declares a transitively-immutable, sendable value is now spelled `imm` and is a reserved token (`KW_IMM`), finishing the ownership-qualifier surface started in #7398 where `own`/`mut` became reserved. `imm` mirrors `mut` as its antonym and, unlike `val`, carries no compile-time-constant baggage from Rust/C# nor the shallow-`const` connotation of the JavaScript that the client backend emits. `val` is no longer special: it reverts to an ordinary identifier, so the previously-required lookahead disambiguation is gone and any binding, parameter, field, or type may once again be named `val`. Code that used the marker must rename `val` → `imm` in ownership-prefix position (`x: imm T`, sendability payloads, and the `E1309` deep-immutable rule); the semantics are unchanged. Internally `OwnershipKind.VAL` is renamed to `OwnershipKind.IMM` (its integer value is unchanged, so existing `.jir` caches remain compatible on that axis), and `jac.spec`'s `ownership_prefix` rule now extracts `"own" | "imm" | "&" "mut"?` instead of the `NAME` placeholder the contextual `val` produced.
