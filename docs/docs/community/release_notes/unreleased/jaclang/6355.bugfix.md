- **Fix: `jac ai --tui` now reaches the plugin agent (hook wiring)**: `JacCmd.run_ai_agent` was declared as a pluggable hook (its docstring says a plugin can register its own implementation to replace the built-in agent) but was never marked `@hookable`, so the `#6856` `@hookable` sweep skipped it. `jac ai` therefore called the built-in byLLM REPL directly and never consulted plugin implementations, which made `--tui` -- and any plugin-registered agent override -- silently ignored. Marking the method `@hookable` routes dispatch through the plugin manager: the jac-super `--tui` launcher now runs for `--tui`, while plain invocations fall through to the default REPL. (jaseci-labs/jaseci#6355)
