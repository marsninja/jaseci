- **Fix: native backend demotes un-lowerable abilities instead of crashing the compile (#6967)**: The `.na.jac` native pipeline is best-effort: an ability it cannot lower is supposed to fall back to its Python implementation, never abort the whole module's native compile. Two paths violated that. (1) Binding a void-typed value to a variable (e.g. `k = none_returning_call()`, or a cross-module call whose native return type collapsed to `void`) asserted deep in `_entry_alloca` -> `PointerType.as_pointer()` on a void pointee; `_codegen_assignment` now emits `E5090` for a void-typed binding so the enclosing ability demotes cleanly. (2) A hard Python exception raised mid-codegen of a method body (e.g. a `void` operand type mismatch in a binary op) propagated out of `_codegen_method` and aborted the entire module, because `_demote_method_if_unlowerable` only fired on emitted diagnostics, not raw exceptions; per-method body codegen is now wrapped so a demotable method converts the crash into `E5090` and falls back to Python, while construction-critical (`init`/`postinit`) and vtable methods still propagate. Together a module with some un-lowerable abilities now natively compiles the rest: the bootstrap lexer lowers 19 methods + 3 functions where it previously crashed with an `AssertionError`. Also quiets the deferred autonative self-acceleration's per-method "native seam" warnings via a thread-local (`JAC_NATIVE_DEBUG` still surfaces them) and documents the `[run] autonative` switch in `jac.toml`.
