- **Fix: rt cache no longer collides across co-located checkouts**: The single-binary runtime cache (`~/.cache/jac/rt/`) was keyed only on the payload content hash, so two `jac` checkouts with byte-identical payloads shared one materialized tree, and the second checkout could silently execute the first's dev-linked `jaclang` source, masking per-branch differences during local testing. The cache dir is now `rt/<hash16>-<pathhash>`, folding in a digest of the binary's own path so each checkout gets its own tree; the shared version prefix still lets stale-tree GC reclaim previous payload versions without evicting co-located siblings.
