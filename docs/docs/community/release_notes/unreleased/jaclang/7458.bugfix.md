- **Fix: `jac fmt --lintfix` no longer turns reassigned `with entry` variables into duplicate `glob` declarations**: the lint-fix pass hoists top-level assignments out of unnamed `with entry` blocks into `glob` declarations, but it promoted every qualifying assignment, including names bound more than once. Since a `glob` is a declaration, a reassigned name was rewritten into multiple conflicting declarations (`x = 1; x = 2;` became `glob x = 1, x = 2;`), and reassigning a name that was already a module-level `glob` merged the reassignment into that declaration. Both misrepresent a reassignment as a redeclaration. A name is now promoted only when the resulting `glob` would be its single module-level declaration; names bound more than once keep all of their bindings inside the `with entry` block. Single-assignment promotion is unchanged.
