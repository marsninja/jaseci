- **Feature: Archetype semantics registered in MTIR**: `MTIRGenPass` now records every archetype's `ClassInfo` (class semstr, `has`-fields, and abilities) in the MTIR map keyed `<module>.<ClassName>`, so the LLM runtime can describe walker/node/edge types (e.g. for `visit [-->] by llm` routing). Previously only `by llm` ability functions had MTIR entries, so walker/node attributes and ordinary `can ... entry` abilities were dropped. Also fixes a latent `ClassInfo` construction crash on the base-class fallback path and removes the now-dead `filter_visitable_by` reflection helpers. (#6767)
