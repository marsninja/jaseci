- **Fix: native calls to bundled stdlib resolve against the bundled signature**: On the native (`na`) pathway, a call to a function or method imported from a bundled `na_stdlib` module was type-checked, keyword-bound, and default-filled against CPython's typeshed stub instead of the bundled `.na.jac` module that is actually linked and run. This broke modules whose signature diverges from CPython (keyword args bound to the wrong slot; omitted params filled from non-lowerable typeshed defaults). Native imports of bundled stdlib modules now resolve to the bundled `.na.jac` signature, so type-check, keyword binding, and default filling all use the linked module.
