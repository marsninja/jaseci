- **Fix: diagnostic help text is now formatted, and borrow expressions type-check**: `DiagnosticInfo.help_text` was rendered verbatim, so templated help strings leaked literal placeholders (e.g. `E1301`'s help showed `'{name}'` instead of the binding name). Help text is now formatted with the same kwargs as the message, guarded so help containing literal braces (such as `E0025`'s `{ ... }` syntax example) falls back to raw. Separately, a borrow expression `&x` / `&mut x` evaluated to `<Unknown>` because the type evaluator's `UnaryExpr` case handled `NOT`/`MINUS`/`BW_NOT` but not unary `&`; it now resolves to the operand's type, eliminating the spurious `E1001`/`E1053` "Cannot assign `<Unknown>`" errors that masked the borrow checker's real `E13xx` diagnostics on every borrow fixture.
- **Fix: a reserved-word local no longer silently disables the whole type checker**: a bare local named `root` in the ownership pass's markers annex shadowed the graph-root keyword and broke codegen for the module, so its meta-import raised "No bytecode found". Because the type-check scheduler swallows that in an `except ImportError` fallback, the failure silently disabled the entire type-check schedule -- every ownership diagnostic (`E1301`-`E1309`) and static analysis went quiet rather than erroring. The local was renamed, restoring the checker.
- **Fix: native cycle collector -- buffered-object reclaim discipline, zero-overhead default, and container tracing (#7149)**: Four deviations from the Bacon-Rajan trial-deletion invariants are closed, all validated by an in-process JIT stress harness (`test_cycle_collector.jac`) that drives cyclic-graph fixtures with cycle collection on and off and asserts on the glibc allocator's in-use byte count. (1) The release path no longer frees an object that is still buffered as a purple cycle-root candidate -- doing so left a dangling pointer the next sweep dereferenced; a buffered object at refcount 0 is left for the collector, which alone may free it. (2) `CollectWhite` now honors the `not buffered` guard from the paper, so a root reached through another root's descent is not freed out from under the still-pending root-buffer iteration (the pass-3 read-after-reclaim). (3) Root buffering is gated on `__rc_cycles_enabled`: in the default cycles-off mode a decremented-but-live object is never recoloured purple or pushed, so the root buffer never allocates or grows -- previously it grew unboundedly for the entire run. (4) Containers whose elements are managed pointers now emit and install a trace function (`__rc_trace_list_*` etc.) so a cycle routed through a container's backing array is reachable by the collector instead of leaking; a participating type that reaches codegen without a trace function is now a compile-time ICE (`E9002`) rather than a silent runtime leak.
