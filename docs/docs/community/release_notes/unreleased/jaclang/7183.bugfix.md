- **Bugfix: literal types inferred from an initializer now widen for mutable variables, attributes, and container elements** (issue #7159): the checker previously kept the `Literal[...]` type when inferring the declared type of a mutable binding, so a later store of a different value failed with `E1001` — e.g. `row = ["."] * 3; row[0] = "B";` reported `Cannot assign Literal["B"] to Literal["."]`. Widening now reaches into container element types (list/dict/set/tuple) and the attribute declaration seams, matching mypy/pyright: `["."] * 3` declares `list[str]` and a game grid `[["." for _ in range(w)] for _ in range(h)]` declares `list[list[str]]`. `Literal` is still preserved for explicit annotations, `Final` bindings, and expression-position inference, so equality/identity narrowing is unaffected.
