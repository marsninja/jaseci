- **Feature: Capability registry for scale dependencies**: Core jaclang gains a single-source-of-truth capability registry (`jaclang.project.capabilities`) that maps each scale capability (`serve`, `data.mongo`, `deploy.k8s`, ...) to its pinned pip deps and a `runtime`/`deploy` context. Capabilities are derived from `[scale]` config intent and the command being run, then resolved into two context-split closures so a runtime pod never carries deploy-only clients like `kubernetes`/`docker`. `DependencyInstaller.sync_capabilities` installs a resolved closure into the project `.jac/venv`.
