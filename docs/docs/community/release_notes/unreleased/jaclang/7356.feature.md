- **Official jaseci base image + automatic pod base selection**: every release now publishes `jaseci/jaclang:<version>` and `:latest` to Docker Hub, and every main push refreshes the rolling `:dev` tag - multi-arch (amd64/arm64), built from the same binaries as the release assets, with jac's runtime pre-extracted and the scale serve closure pre-installed. Scale deploys pick the image automatically by channel (dev -> `:dev`, stable -> `:latest`) when it is published and `[scale.kubernetes].python_image` is unset, falling back to `python:3.12-slim` otherwise; pod bootstraps reuse the image's baked binary when it is byte-identical to the shipped one, skipping the ~2-minute per-pod runtime extraction (measured on EKS: fleet serving in 249s vs 379s, gateway Ready +71s vs +181s). CI/CD setup: add two repo secrets, `DOCKERHUB_USERNAME` and `DOCKERHUB_TOKEN` (an access token from the `jaseci` Docker Hub org with write access); the `docker-image` job skips quietly when they are absent.
