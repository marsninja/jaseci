- **Fix: native mixed-type ternary emits valid IR (#6987)**: A ternary whose two branches needed a numeric coercion (e.g. `1.5 if c else 2`, or two different int widths) miscompiled on the native backend. `_codegen_if_else_expr` positioned the builder at the merge block and only then coerced the branch values, so the `sitofp` / `zext` cast was emitted into the merge block *ahead of* the `phi`. LLVM requires every PHI to be among the first instructions of its block, so the verifier rejected the module ("PHI nodes not grouped at top of basic block") and native compilation failed, even though the Python backend handled the same code fine. Each coercion cast is now emitted in its predecessor block, just before that block's branch to the merge block, leaving the phi at the top of the merge block, which mirrors how the binary-op and short-circuit boolean paths already coerce operands before branching. Same-type branches emit no cast and were never affected.
