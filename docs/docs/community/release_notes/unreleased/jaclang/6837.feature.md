- **Feature: access-modifier enforcement (`:pub` / `:protect` / `:priv`)**: a new `AccessCheckPass` (in the type-check schedule) enforces the access tags along two axes, chosen by where a symbol is declared. For members (`has`/`def` inside an archetype): `:pub` is accessible anywhere, `:protect` in the declaring class and its subclasses, `:priv` in the declaring class only. For top-level symbols (`glob`/`obj`/`def`/`enum`/`walker`): `:pub` is visible from any module including a consuming project (an exported library endpoint, so it need not be referenced inside its own project), `:protect` is visible within the same project (the directory tree rooted at its `jac.toml`), and `:priv` is visible only in the declaring module. Violations are warnings by default and promoted to hard errors with `[check] enforce_access = true` in `jac.toml`. New diagnostics `E/W2034` (private member), `E/W2035` (protected member), `E/W2037` (module-private cross-module), and `E/W2038` (project-protected cross-project) are individually suppressible via the existing `[check] suppress` / `# jac:ignore[...]` mechanisms. Endpoint/`:pub` behavior (`jac serve`, eject auth) is unchanged -- these rules govern call/reference legality, not endpoint surfacing.
- **Fix: `glob :priv` / `glob :protect` access modifier was dropped**: a module-private or project-protected global had its access silently reset to public because the assignment target was bound before the access-carrying global-var insert; module-private globals now enforce correctly.
