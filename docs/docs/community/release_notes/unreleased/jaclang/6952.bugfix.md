- **Fix: `-Ddev` binary no longer bricked by `JAC_NO_DEV_SOURCE`**: a linked `zig build -Ddev` binary (#6948) ships no bundled jaclang, so the baked `jac_linked_source` marker is its only compiler. `apply_dev_source_override` now resolves that marker *before* the `JAC_NO_DEV_SOURCE` guard, so the flag (which means "use the shipped compiler, not a dev tree") no longer leaves a linked binary with no source on `sys.path` and a crash on the first `.jac` import; `JAC_NO_DEV_SOURCE` continues to suppress only the `jac.toml`-based dev loop. Also caps `mkpayload`'s argv collection at the array size so a 17+ argument invocation can't index past the fixed buffer.
