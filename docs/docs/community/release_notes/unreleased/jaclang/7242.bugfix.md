- **Fix: Tuple destructuring and per-index typing on the native pathway**: `for (name, size) in files` now lowers on `.na.jac` — a tuple-target for-loop over a list binds each name from the element's per-position field (reusing the existing tuple-unpack path) instead of dying at codegen with E5090. In the type checker, literal-index access on a fixed heterogeneous `tuple[T1, T2, ...]` now types per position (so `f[1]` on `tuple[str, bytes]` is `bytes`, negatives supported) rather than collapsing to the first element type, and destructuring a variadic `tuple[T, ...]` types every target as the element type instead of leaking the `...` placeholder.
