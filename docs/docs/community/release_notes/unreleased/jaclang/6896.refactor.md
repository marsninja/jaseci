- **Single-binary cleanup**: every install (`jac install`, `jac install <pkg>`, `jac install -e <path>`, `jac add`) targets the current project's `.jac/venv` -- third-party packages live only there, never in a binary-global or host environment. `jac install <pkg>`/`-e` therefore require a `jac.toml` (they error outside a project); `-e` reads the target package's deps from its own `jac.toml` but installs and links them into the current project's venv. The binary discovers a project's `.jac/venv` at startup and gives it `sys.path` precedence, so per-project deps and editable plugins shadow the bundled site. Removes leftover editable-install/packaging scaffolding (`setup.py`, `jaclang.pth`, `MANIFEST.in`) now that jaclang ships only as the native `jac` binary; `sitecustomize.py` is kept but rewritten jaclang-free to wire the project venv and the lazy `.jac` import finder.
