- **Single-binary cleanup**: `jac install <pkg>` and `jac install -e <path>` install into the binary's activated environment (the global/tool-install path that works outside a project); per-project deps go through `jac add` / the no-arg `jac install`, which target the project's `.jac/venv`. The binary discovers a project's `.jac/venv` at startup and gives it `sys.path` precedence, so per-project deps and editable plugins shadow the bundled site. Removes leftover editable-install/packaging scaffolding (`setup.py`, `jaclang.pth`, `MANIFEST.in`) now that jaclang ships only as the native `jac` binary; `sitecustomize.py` is kept but rewritten jaclang-free to wire the project venv and the lazy `.jac` import finder.
