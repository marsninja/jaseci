- **Fix: erroring native modules no longer execute silently (check/run parity)**: a `.na.jac` module with native codegen errors (`E5090` and friends) previously printed the errors once, then published the holey LLVM IR, Python bytecode, and a poisoned `.jir` cache entry anyway. Importing scripts executed the module with the erroring expressions elided, warm-cache runs succeeded silently with exit 0, `jac test` swallowed the diagnostics behind `ImportError('No bytecode found')`, and `jac check` passed the file clean. Now: `NaIRGenPass` withholds IR from an erroring module, `compile()` scrubs all runnable artifacts (never caching a poisoned `.jir`), imports fail with an `ImportError` carrying the pretty-printed diagnostics (so `jac test` reports the real compile failure like pytest does a `SyntaxError`), and `jac check` runs the native codegen pass itself for native content so it reports exactly what `jac run` rejects. Method seam demotion to a full Python implementation (#6967) is unaffected. Also fixes an `E1004` false positive: a function ending in `while True` with no `break` can only exit via return/raise and is no longer flagged as implicitly returning `None`.
