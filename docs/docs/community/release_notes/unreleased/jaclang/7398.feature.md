- **Feature: emit-time `--gc` mode selection and RC-elision coverage stat (#7398)**: `jac nacompile --gc <mode>` (and `CompileOptions.na_gc_mode`) chooses the memory-management runtime the native backend emits: `cycles` (default -- reference counting plus the opt-in Bacon-Rajan cycle collector, unchanged from #7149), `rc` (plain reference counting with no collector functions, trace functions, roots buffer, or `JAC_GC_CYCLES` entry probe emitted at all), or `none` (no retain/release call sites emitted -- the compile-time analogue of `JAC_NO_GC`, where heap memory is never reclaimed). Separately, compiling with `JAC_RC_STATS=1` prints a per-module line reporting retains and releases emitted versus reference-count operations elided by move-lowering, e.g. `rc-stats [mod.na.jac] gc=cycles retains=1 releases=10 elided=3 coverage=21.4%`.
