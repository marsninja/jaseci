- **Fix: Multiple `with entry` blocks in a native module (#6660)**: `jac nacompile` on a `.na.jac` module with two or more module-level `with entry { ... }` blocks crashed during IR generation with `llvmlite.ir._utils.DuplicatedNameError: jac_entry`, because the native backend emitted a separate `jac_entry` function per block. The blocks are now concatenated, in source order, into a single `jac_entry` (with the glob-init / `JAC_NO_GC` / imported-module init preamble emitted exactly once), mirroring the Python target. Single-entry modules are unchanged. As a guard against the whole class of bug, every fixed-name native runtime singleton (`jac_entry`, `__jac_glob_init`, the `__rc_*` / `__jac_exc_*` / `__osp_*` helpers) is now defined through a single `_define_function` helper that raises a clear internal error naming the symbol if an emitter is reached twice, instead of an opaque llvmlite `DuplicatedNameError`.
