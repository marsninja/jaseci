- **Fix: fence heap-element container writes under `--gc none` enforcement**: under `--enforce-nogc --gc none`, owned containers of heap elements are deferred and meant to be E1406-fenced, but the fence only scanned list literals. Heap elements reaching a container via `append`/`add`/`insert`, an index-assigned dict/set value, a dict/set literal, or a freshly-built (owned) string slipped through to headerless codegen, where the container drop helper frees only the backing array — never the entries — leaking the elements (drop hooks silently skipped) or leaving a dangling pointer. The fence now rejects every such write, gated on the receiver statically typing to `list`/`set`/`dict` so user types that merely define an `append` method are not falsely flagged; scalar containers and lists of string literals stay supported.
