- **Fix: clean error for `root`/`super` in every binding position**: `root` and `super` lower to calls (`Jac.root()`/`super()`), so they cannot name a binding. Assignment and parameter names already reported a clear error, but `for root in ...`, comprehension targets, `has root: int`, `with ... as root`, `del root`, and `root := ...` previously leaked a cryptic backend `E5043`. These now all report `E0024` ("Cannot bind 'root' -- it is a built-in reference"), with backtick escaping (`` `root ``) remaining the documented way to bind a local of that name. The check is centralised in one validation-pass helper and recurses tuple/list destructuring; the bindable special names (`self`/`here`/`visitor`/`props`) are unaffected.
