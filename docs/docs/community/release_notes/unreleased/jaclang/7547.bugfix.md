- **Fix: ownership-transfer miscompiles under `--enforce-nogc --gc none`**: headerless codegen dropped values whose ownership had already transferred, in two shapes that rc/cycles masked because retains/releases balance there: reassignment through a consuming call (`x = f(x)`) released the old value the callee had taken, and owned temporaries/locals passed to `own` parameters (or archetype constructors) were released by caller-side call-arg and scope-exit cleanups -- both use-after-free in the emitted binary. The ownership checker now stamps consumed-by-RHS overwrites, and call lowering skips releases for transferring argument positions and nulls the source slot at the move site. Also: E1402 now names the sealed value instead of the storage container; `list[float]`-style return annotations no longer trip the container fence; path expressions (`prog[pc]`) classify by their own type instead of their root's; scalar reads off an owned root (`v = xs.pop()`) no longer consume the owner; and `len(x)` in a loop header no longer raises a bogus E1301 (the loop-move heuristic now shares the builtin exempt list). The W3003 entry-to-glob lintfix no longer hoists call-bearing initializers or names written inside entry-block control flow, which reordered side effects on the native backend where glob initializers all run before any entry block.
