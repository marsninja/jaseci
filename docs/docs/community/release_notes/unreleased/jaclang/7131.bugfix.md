- **Fix: in-pod KubernetesTarget calls no longer die with `NameError: name 'ConfigException' is not defined`**: `kubernetes_target.jac` used `except ConfigException` in nine places without importing the name from `_optdeps.kubernetes`. From a machine with a kubeconfig the except clause was never evaluated (`load_kube_config()` succeeds), so the bug only surfaced inside cluster pods: with no `~/.kube/config`, `load_kube_config()` raises, the except clause hits the NameError, and every in-pod KubernetesTarget call (deploy/destroy/get_status/...) fails before it can fall back to `load_incluster_config()`. This broke operator-style apps that drive deploys in-process from a pod, such as jacBuilder's deploy-to-production. `ConfigException` is now imported alongside `client`/`config`/`ApiException`; the `_optdeps` re-export keeps it optional-dependency-safe.
