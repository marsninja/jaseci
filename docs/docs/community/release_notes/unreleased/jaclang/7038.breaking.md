- **Breaking: `jac-scale` folded into core as `jaclang.scale`**: The separate `jac-scale` PyPI package is gone (clean break). Scale (serve / deploy / microservices) is now a built-in plugin shipped inside the `jac` binary and registered automatically (no `jac install jac-scale`, no `jac plugins enable scale`). Imports move from `jac_scale.*` to `jaclang.scale.*`. Its optional heavy dependencies (fastapi, pymongo, redis, kubernetes, ...) are never bundled in the binary; they are pulled per-project into `.jac/venv` from `[scale.*]` config intent via the capability registry on `jac install`. `jac start` / `jac start --scale` and all `[scale.*]` / `[plugins.scale.*]` config are unchanged. `import jaclang` / `jac --version` import none of the scale runtime deps.
