- **Formatter comment handling rearchitected (clean break from post-hoc injection)**: comments are now attached to AST nodes before doc generation (`CommentAttachPass`) and emitted by the doc generator itself, replacing the deleted `CommentInjectionPass` that re-walked the DocIR DAG as a tree (exponential `2^k` re-walks on nested shared `IfBreak` branches: a 20-deep nested list took 93s and 106M function calls to format; it is now linear). The doc IR gains `LineSuffix` (defers trailing `#` comments to end-of-line, so they land after commas/semicolons by construction) and `BreakParent` (forces enclosing groups to break) primitives. Correctness wins inherited from the new architecture: inline comments inside constructs that previously printed flat are no longer silently deleted (`[1,  # keep me` survives formatting), comment placement is byte-idempotent on comment-dense JSX (regression test ported from #7495), and comments inside empty bodies now indent to body level. The `E5051` refuse-to-save contract is kept via a post-render preservation guard in `JacFormatPass`.
