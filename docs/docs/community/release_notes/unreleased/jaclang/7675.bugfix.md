- **Fix: a plain client import of a server module silently rendered a blank page**: client code that imported a server module with plain `import from` instead of `sv import from` emitted an ES import of a module the client bundle never exports, so the page went blank at module link time with a clean `jac check` and an empty browser console. The ES backend now infers the `sv` boundary from the target's placement: imported server `def:pub` endpoints bridge over RPC exactly like `sv import`, server types follow `sv` type semantics, and importing a symbol with no client-side presence is a new `E5082` compile error, reported by both `jac check` and the client build.
