- **Fix: No false "may be undefined" for ambient typing names in `.impl.jac` files**: ambient typing names such as `Callable` and `Sequence` are injected into the module's parent (builtins) scope by the type evaluator, but `_check_undefined_names` never consulted that scope, so referencing them in `.impl.jac` annex files raised a spurious W2001. The undefined-name check now skips any name resolvable in the module's parent scope, so ambient typing names resolve without an import in both the declaration signature and the impl body.
