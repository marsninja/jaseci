- **Native: foreign-typed fields lower to opaque null slots**: A native (`na`) `has` field whose declared type the backend cannot model (a foreign Python type such as `ast3.AST | None` or `type[Transform]`) now lowers to a nullable opaque slot (null at native construction) instead of rejecting the declaration with E5090; a type-only foreign import that merely names such a type is tolerated, while a genuinely unknown type name still reports E2018 and any actual dereference of the slot still reports E5090.
