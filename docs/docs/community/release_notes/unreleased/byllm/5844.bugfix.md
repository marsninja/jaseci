- **Fix: schema-hint walker recurses into array items**: The universal `inject_schema_hint` from #5837 only descended into `object -> properties`, so `list[Ingredient]`-style returns (where the enum-typed `unit` field lives at `schema.properties.schema_object_wrapper.items.properties.unit`) emitted no hint at all. Local-style backends fell back to the always-pick-first-allowed-integer baseline, so every ingredient came back with `unit = PIECE` regardless of meal. The walker now traverses `array -> items` as well as `object -> properties` and emits bullets only for enum properties (non-enum scalars and container types stay quiet so the hint reads as actionable choices, not a redundant restatement of the schema). Empirical impact on `day_planner/auth`'s `generate_shopping_list` with Gemma 4 E4B: before fix every item was `PIECE`; after fix the model picks `cup` for liquids/grains, `lb` for meat, `tsp`/`tbsp` for seasonings, `piece` for whole items -- six of the seven Unit values exercised across three sample meals, and cost estimates jumped from `$0.00` defaults to realistic per-item prices as a side effect of the better grounding.
