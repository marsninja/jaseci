- **Fix: `run_snippet` output is capped at `MAX_OUTPUT_SIZE`**: `run_snippet` already bounded input size but captured stdout/stderr into unbounded buffers, so a snippet emitting output rapidly could exhaust server memory before the timeout fired. Output is now read in background threads with the same 102400-byte cap as input; excess is drained so the child process is not blocked on a full pipe, and truncated output carries an `[... output truncated ...]` marker.
