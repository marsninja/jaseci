- **Signal-based reactive runtime**: The client runtime now depends on `@preact/signals-react` and re-exports `useSignal`, `useSignalEffect`, and `peek` through `@jac/runtime`. `has` fields in client code become mutable signal cells whose `.value` is always live across sync, async, and effect boundaries -- matching Jac's server-side `has` semantics. The `@preact/signals-react/auto` integration is bundled, so JSX reads of `signal.value` subscribe the enclosing component to re-render without any user setup. `useState` is no longer re-exported from `@jac/runtime`; user code that wants React's `useState` should import it directly from `react`. `peek()` is provided as the escape hatch for reading a signal without subscribing an effect to it.
