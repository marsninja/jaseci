- **Feature: Framework-agnostic web codegen via `FrameworkBackend` seam**: `.cl.jac` / `cl {}` codegen is no longer hard-wired to React. A `FrameworkBackend` interface + neutral `reactive_intent` / `view_ir` layers in `EsastGenPass` delegate reactive and view lowering to a pluggable backend. Backend is selected via `[client] framework` in `jac.toml` (default `"react"`). Unknown values raise `E5081`.
- **Feature: Preact backend**: `PreactBackend` wires `preact/compat` aliases so existing React runtime and `JacForm` code runs under Preact with no source changes. Includes compiler smoke test and E2E coverage.
- **Feature: Solid backend (View IR Branches 1–5)**: `SolidBackend` emits JSX in the dom-expressions dialect, uses native `<For>` / `<Show>` control-flow components, `@solidjs/router` file-based routing, and `vite-plugin-solid`. `solid_runtime.cl.jac` ships a Solid-flavoured layout root and `JacForm` shim. jsdom gate passes 5/5 (counter, routing, control flow, full `JacForm`, P2 dynamic-prop / `ref` e2e / callback-prop matrix).
- **Refactor: TanStack Form replaces React Hook Form (complete)**: `@tanstack/react-form` and `@tanstack/solid-form` replace RHF across all runtimes. Unified `onChange`-only `onTouched` mode on all backends (TanStack #1784 workaround). `effectiveIsValid` submit gate, import guards, `CORE_RUNTIME_SYMBOLS` manifest, tarball publish scans, and strict peer declarations included.
- **Fix: JacForm `onTouched` error display**: Errors are now suppressed until a field is touched (blurred or changed) in `onTouched` mode, matching the expected UX.
- **Fix: JacForm inputs missing `name` attribute**: All `renderInput` variants (text, number, email, password, checkbox, select, radio, textarea) now emit `name={fieldName}`, restoring native form submission, `<label for>` association, accessibility, and Playwright `input[name=...]` selectors that were silently dropped in the TanStack Form migration (RHF's field spread carried `name` automatically; TanStack does not).
- **Tests: Playwright form E2E**: `test_form.jac` covers error DOM display, submit-button disabled → enabled transition, and `onSubmit` callback firing.
